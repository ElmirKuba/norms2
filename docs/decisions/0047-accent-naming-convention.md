# ADR-0047: Конвенция имён раздела «Акцент» (терминология)

- Статус: принято (2026-06-15, фаза 2 / R0–R1 вычитка доки)
- Контекст: дока «Акцента» синтезирована из источников вдохновения (SuperBetter,
  Ascent) с **англо-геймерскими именами**: Epic Win, PowerUp, BadGuy, Ally,
  Secret Identity, Quest. Для русского, свободного/бесплатного «для своих» продукта
  с тоном «интерес, не страх» (README §1/§6) этот пафос неуместен и местами
  инфантилен. Нужна **единая конвенция имён** — чтобы код, БД, API-routes, UI-копирайт
  не расходились. Опора — конвенции фазы 1.

## Развилка (ToT)

| Ветвь | Суть | Минус |
|---|---|---|
| A. Оставить геймерские имена | Epic Win/BadGuy… в коде и UI | пафос/инфантильность; не наш тон; англицизмы в UI |
| B. Русифицировать всё (вкл. код) | имена сущностей по-русски | против нормы индустрии; кириллица в идентификаторах кода/БД |
| **C. Нейтрально-EN код + RU UI** | код — нейтральный английский, интерфейс — русский | нужно один раз провести маппинг |

## Решение

**Ветвь C.** Код/БД/API/компоненты — **нейтрально-английские** имена (норма индустрии);
**UI** — русский; геймерский пафос (Epic/Secret/BadGuy) выкинут. Маппинг:

| Источник | Код (сущность/таблица/route) | UI (рус.) |
|---|---|---|
| Epic Win | `Goal` / `goals` / `/accent/goals` | «Цель» |
| PowerUp (+Log) | `MicroWin` (+`MicroWinLog`) / `micro_wins` / `/accent/micro-wins` | «микро-победа» |
| BadGuy | `Obstacle` / `obstacles` / `/accent/obstacles` | «препятствие» |
| Ally | `Supporter` / `supporters` / `/accent/supporters` | «поддержка / союзник» |
| Secret Identity | `Identity` / `accent_identities` | «личность» |
| Quest (разовая) | `Task` (one-off, `templateId=null`) | «разовая задача» |
| (источник очков/событие) | `micro_win` / `micro_win.completed` | — |

AntiHabit / CheckIn / Streak / Goal / Habit / Task / WeeklyGoal / DailyLesson —
уже нейтральны, остаются. Компоненты фронта по тому же маппингу (`MicroWinChip`,
`CheckinSlider`…).

## Последствия

- Применено в **R1a** ко всем 5 файлам доки `docs/sections/accent/` (вычитка, grep-чисто).
- При кодинге фазы 2 сущности/таблицы/routes/DTO/компоненты **следуют маппингу**;
  отступления — только осознанные и через обновление этого ADR.
- Исторические упоминания старых имён в доке допустимы только как пометки «бывш. …»
  (объяснение решений), не как действующие идентификаторы.

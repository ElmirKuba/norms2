## 1. Контекст и мотивация

В сети распространены платные шаблоны Google Sheets для геймификации жизни: недельные цели, чеклисты на каждый день, графики прогресса, "лучший день / худший день", "урок дня", метрики сна/энергии/настроения. Шаблон — это статичная таблица, без серверной агрегации, без напоминаний, без серий, без достижений, без mobile-friendly интерфейса, и без долгосрочных целей с дедлайном (формат `00/77 (03.05.2026)`).

**Ascent** воспроизводит идею шаблона как полноценный сервис и расширяет её:

1. **Долгосрочные цели** с измеримым прогрессом (`X / N`) и дедлайном — заимствовано из личных заметок ("Бег: 00/77 (03.05.2026)", "Борьба с дешёвым дофамином: 00/77").
2. **Категории "три победы"** — духовная, физическая, ментальная. Каждая цель принадлежит одной из категорий, дашборд показывает баланс.
3. **Недельный планер** — 5 целей на неделю, дневные чеклисты, метрики самочувствия, "урок дня", агрегация по неделе (лучший/худший день, средний %).
4. **Анти-привычки** — отдельный реестр того, что НЕ делать (не есть треш, не смотреть порно). Учитываются как инверсные серии (streak без срыва).
5. **Запланированные действия** — медицинские анализы, разовые задачи с напоминаниями (тестостерон, эстрадиол, пролактин).
6. **Геймификация** — очки, серии (streaks), уровни, достижения, визуальный фидбэк на каждое выполнение.

Проект **личный, self-hosted**, без претензий на multi-tenant SaaS, но с возможностью использования несколькими пользователями (семья, друзья) на одном инстансе.

## 2. Принципы

1. **Измеримость** — любая цель сводится к счётчику с числителем и знаменателем. Никаких "стать лучше". Чёткое `20 приседаний`, `пробежка 5 км`, `7 дней подряд без сахара`.
2. **Серверная истина** — все агрегаты (проценты, серии, лучший день) считаются на бэке. Фронт только отображает. Никаких рассинхронов между устройствами.
3. **Идемпотентность** — отметить задачу можно несколько раз, результат тот же. Снять отметку — отдельный явный сигнал.
4. **Темпоральность** — у каждой записи есть дата (день в TZ пользователя). Перенос задачи на следующий день — явная операция, а не редактирование.
5. **Шаблоны и инстансы** — `TaskTemplate` (повторяющаяся привычка) и `Task` (её инстанс на конкретный день) — разные сущности. Нельзя редактировать историю задним числом неаккуратно.
6. **Open in browser, ready for mobile** — вёрстка mobile-first, но web-only на старте. Capacitor-обёртка — отдельная фаза, не блокирующая запуск.

## 4. Доменная модель (обзор)

Полные определения, инварианты и связи — [`docs/domain-model.md`](./docs/domain-model.md). Здесь — карта.

```
User
  ├─ Goals (долгосрочные, X/N, дедлайн, категория: spiritual|physical|mental)
  │    └─ GoalEntries (инкрементальные записи прогресса с датой)
  ├─ TaskTemplates (повторяющиеся привычки: "встать в 6 утра", "зарядка")
  │    └─ Tasks (инстанс на конкретный день с состоянием)
  ├─ AntiHabits (что НЕ делать: "не ем треш", "не смотрю порно")
  │    └─ AntiHabitDays (на каждый день отметка: устоял / сорвался)
  ├─ WeeklyGoals (5 целей на неделю)
  ├─ DailyMetrics (сон, энергия, настроение)
  ├─ DailyLessons (урок дня — рефлексия)
  ├─ Workouts (программы №1..№5 — шаблоны тренировок)
  │    └─ WorkoutSessions (выполненная тренировка)
  ├─ ScheduledActions (разовые задачи с дедлайном: "сдать тестостерон")
  └─ Achievements (автоматически выдаваемые)
```

Ключевые инварианты:
- **TaskTemplate** определяет правило повторения (RRULE-подобное); **Task** материализуется на конкретный день при ролловере.
- **Goal** имеет два числа: `current` и `target`. Прогресс ≥ 100% не означает удаление цели — она просто переходит в `completed` со временем выполнения.
- **AntiHabit** ведёт себя инверсно: серия растёт за каждый день без срыва; срыв сбрасывает серию, но не очки.
- **Streak** считается по факту выполнения подряд (или по факту устояния — для AntiHabit), вычисляется на бэке.

## 5. Функциональные требования

### 5.1 Цели (Goals)

- Создать цель: название, категория (`spiritual` / `physical` / `mental`), `target` (число), единица измерения (повторения / км / дни / страницы), дедлайн (опционально).
- Добавить запись прогресса (`GoalEntry`): сколько единиц добавлено сегодня (или в указанную дату). Бэк суммирует.
- Список целей с фильтром по категории и статусу (`active` / `completed` / `archived`).
- Карточка цели: прогресс-бар, оставшиеся дни до дедлайна, средний темп `units/day`, прогноз "успею ли" (линейная экстраполяция).
- Архивация и восстановление.

### 5.2 Шаблоны задач и дневной планер

- CRUD `TaskTemplate`: название, описание, иконка, RRULE (ежедневно / по дням недели / N раз в неделю / каждые N дней).
- Ролловер: каждый день в 00:00 TZ пользователя бэк создаёт `Task` для каждого активного `TaskTemplate` на сегодня.
- Просмотр дневного плана: список задач на дату, чекбоксы.
- Toggle задачи: одной кнопкой выполнить / снять. Время фиксируется.
- Перенос задачи на завтра (одна кнопка) — создаёт инстанс на следующий день, текущий помечает `skipped` с причиной `postponed`.
- Добавление разовой задачи на день (без шаблона).

### 5.3 Анти-привычки

- CRUD `AntiHabit`: название, описание.
- На каждый день — состояние: `held` (устоял), `relapsed` (сорвался), `unset` (не отмечено).
- Серия `consecutive_days_held` — главный показатель.
- История с календарной heatmap.

### 5.4 Недельный планер

- 5 (или N — настраивается) целей на текущую неделю — короткие чек-айтемы.
- Прогресс по неделе: сумма выполнения за пн—вс, лучший день (самый высокий процент), худший день, средний процент.
- Bar chart выполнения по дням (как в исходном шаблоне).
- При начале новой недели предыдущая архивируется.

### 5.5 Метрики дня (`DailyMetrics`)

- Сон: часы (число с шагом 0.5).
- Энергия: 1..10 (слайдер).
- Настроение: 1..10 (слайдер).
- Заметка к самочувствию (опционально).
- Графики динамики за 7 / 30 / 90 дней.

### 5.6 Урок дня

- Свободный текст (markdown), тегирование.
- Индекс по тегам, поиск.

### 5.7 Тренировки

- `Workout` — шаблон: название (например, "Грудь / трицепс"), список упражнений (имя, подходы, повторы, вес — последний known).
- `WorkoutSession` — выполненная тренировка: ссылка на шаблон, дата, фактические подходы/повторы/вес, RPE.
- В дневной плане можно "запустить" тренировку, чекбоксы по упражнениям.

### 5.8 Запланированные действия

- Разовые задачи с дедлайном и напоминанием (в `n` дней до).
- Список открытых, история выполненных.

### 5.9 Геймификация

- Очки за каждое выполнение (вес зависит от типа сущности — детали в [`docs/gamification.md`](./docs/gamification.md)).
- Уровень = функция от суммарных очков.
- Серии: `current_streak`, `longest_streak` для каждой задачи / привычки и общая.
- Достижения: автоматически выдаются по триггерам ("7 дней подряд встаю в 6 утра", "первая цель закрыта", "месяц без срывов").

### 5.10 Дашборд

Главный экран. На одном экране:
- "Сегодня": дневной план (чеклист), общий процент выполнения, текущая серия.
- Недельный прогресс: donut + bar chart по дням.
- Активные цели: 3–5 карточек с прогрессом.
- Метрики дня: сон / энергия / настроение (быстрый ввод).
- Уведомления о просроченных запланированных действиях.

UI вдохновлён референсом из шаблона (тёмная тема, бежевые акценты), детали — [`docs/ui-ux.md`](./docs/ui-ux.md).

## 6. Нефункциональные требования

| Категория | Требование |
|---|---|
| Производительность | Дашборд за < 300 мс на P95, 1 пользователь, локальный сервер |
| Доступность | Self-hosted, single-node; HA не требуется |
| Безопасность | bcrypt для паролей, JWT с коротким TTL, refresh с rotation, HTTPS обязателен в проде |
| Приватность | Данные не покидают инстанс пользователя. Никакой телеметрии |
| Локализация | RU / EN, переключатель в профиле. Даты и числа через Intl |
| TZ | Каждый пользователь имеет TZ; ролловер дня и сериал по локальной полуночи |
| Доступность (a11y) | WCAG AA по контрасту, навигация с клавиатуры, ARIA-роли |
| Резервные копии | Скрипт `pg_dump` + ротация, описано в [`docs/deployment.md`](./docs/deployment.md) |

## 10. Критерии готовности MVP

MVP считается готовым, когда:

1. Пользователь может зарегистрироваться, залогиниться, выйти.
2. Создать долгосрочную цель и инкрементировать её прогресс.
3. Создать шаблон привычки, увидеть её в дневном чеклисте, отметить выполнение.
4. Ввести метрики самочувствия за день.
5. Видеть на дашборде: сегодняшний план, недельный прогресс с лучшим/худшим днём, активные цели.
6. Получить хотя бы одно достижение и увидеть его в профиле.
7. Развернуть всё через `docker-compose up` на чистой машине по инструкции из [`docs/getting-started.md`](./docs/getting-started.md).

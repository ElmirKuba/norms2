# ADR-0024: Cookie-гейт при входе (B4)

- **Статус:** accepted
- **Дата:** 2026-05-30
- **Решает:** Elmir (пункт B4)
- **Контекст-теги:** [frontend] [152fz] [ux]

## Решение
Блокирующий cookie-гейт при первом визите:
- Текст: «куки нужны для корректной работы сайта» + ссылка на страницу согласия/политики.
- Кнопки: **«Я согласен»** (ставит флаг согласия, пускает в приложение) и **«Покинуть сайт»**.
- Без согласия контент SPA не рендерится — **кроме `/privacy`** (политику логично прочитать ДО «Я согласен»): этот путь доступен и без согласия, остальное гейтится. Реализовано в `App` (`showRoutes` = согласие ИЛИ url начинается с `/privacy`).
- Флаг согласия — в `localStorage`.

## Честные оговорки (зафиксировано осознанно)
1. **Юридически** наши cookie — только технические/auth (без аналитики/ПДн), и для них согласие **не обязательно**. Гейт — продуктовый/эстетический выбор «системы для своих», а не требование закона.
2. **«Покинуть сайт» не может гарантированно закрыть вкладку:** `window.close()` работает только для вкладок, открытых скриптом. Поэтому кнопка делает **redirect на нейтральную/прощальную страницу** (или внешний сайт), а не полагается на закрытие вкладки.

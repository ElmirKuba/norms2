# ADR-0056: Микро-победы — две оси классификации (модальность + сфера)

- Статус: принято (2026-06-29, фаза 2, подфаза 2.5·28, реш. Elmir — выбор «D+C») — **обратимо**
  (`domainKey` опционален и аддитивен; можно перестать показывать поле без потери данных).
- Контекст: находка **M#B3-1** при регрессе ·28.B3. Микро-победы классифицировались полем
  `category` (11 значений: Телесное/Ум/Эмоции/Общение/Сенсорика/Быт/Цифровое/Отдых/Тишина-смысл/
  Природа/Границы), а цели и привычки — полем `domainKey` (6 «сфер жизни» из `accent_domains`:
  Сила/Дисциплина/Интеллект/Дух/Социальность/Здоровье и т.п., наполняется сидом). Возникал вопрос:
  это рассогласованная одна классификация — свести к одной? Анализ (3 шляпы) показал: **это разные
  оси**, а не рассинхрон.

## Решения
1. **Категория и сфера — РАЗНЫЕ оси, обе остаются.**
   - **`category` (модальность, обязательна)** — «какой это сброс»: подвигаться / успокоиться / уйти
     с экрана / отдохнуть. Отвечает на потребность в плохой день; ось восстановительного действия.
   - **`domainKey` (сфера, опциональна)** — «какую область жизни питает», общая ось со целями/
     привычками. Включена ради будущей агрегации «баланс сфер» (дашборд 2.11+) и единой ментальной
     модели — БЕЗ навязывания achievement-рамки (поле необязательное; микро-победа остаётся про «не
     достигать, а восстановиться»).
2. **Рассмотренные альтернативы (ToT):**
   - **B (заменить 11 категорий на 6 сфер)** — отвергнута: breaking + миграция с потерей информации
     (Цифровое/Отдых/Природа/Границы некуда лечь) + философски навязывает достижение инструменту НЕ
     про достижение.
   - **A (оставить как есть, только задокументировать)** — частично взята (см. п.4).
   - **D+C (выбрано):** развести коллизию ярлыков (**D**) + добавить опц. сферу второй осью (**C**).
3. **`domainKey` у микро-побед — мягкий ключ (varchar 64, nullable, без FK)**, как у целей/привычек
   ([ADR-0048]). Сквозь все слои (schema+миграция 0022, DTO, domain-service, repository, view). Форма
   получила опц. select «Сфера» из того же справочника `accent_domains`.
4. **D — развод коллизии ярлыков:** категория `spiritual` «🌱 Дух / смысл» → «🧘 Тишина / смысл»
   (читалась идентично сфере «Дух»); описание уточняет «это вид действия, не сфера жизни». Остальные
   ярлыки коллизий не дают.

## Последствия
- Микро-победа несёт обе классификации; пользователь обязан выбрать модальность и **может** указать
  сферу. Простота формы плохого дня сохранена (сфера — опциональна, в самом низу).
- Открыт путь к кросс-разделовой агрегации «баланс сфер» (цели+привычки+микро-победы) в дашборде
  (2.11+) — но это **не реализуется сейчас** (нет потребителя).
- Стартовый пак микро-побед пока без `domainKey` (null) — можно проставить позже отдельным сидом.
- Связано: [[0048-accent-domain-refinements]], [[0052-accent-goal-direction-and-computed-progress]],
  [[0053-goal-focus-and-mission-filter]]; находка M#B3-1 в `impl-phase2-plan.md`.

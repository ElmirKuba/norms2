## A. Создание целей — 4 рода
- [ ] **A1 accumulate** — «Накопить», единица, цель >0 → создаётся; бейдж «Накопить».
- [ ] **A2 reach** — «Достичь уровня», цель ≠ старт → создаётся; есть поле «Старт».
- [ ] **A3 reduce** — «Снизить», цель ниже старта → создаётся.
- [ ] **A4 maintain** — «Удерживать», **нижняя < верхняя** → создаётся; бейдж «Удерживать».
- [ ] **A5 валидации:** accumulate цель=0 → ошибка; reach цель=старт → ошибка; maintain без нижней / ниж≥верх → ошибка (понятный текст, форма не закрывается).
- [ ] **A6 переключение рода в форме** — выбираешь reach → появляется «Старт»; maintain → «Нижняя/Верхняя граница»; accumulate → подсказка-миссия + «ради чего откажусь». Поля реагируют сразу.

## B. Запись прогресса + история (реактивно, без F5)
- [ ] **B1 accumulate** — «+N» → процент/амаунт растут сразу; запись в истории.
- [ ] **B2 reach/reduce** — замер → текущее = последний замер; процент пересчитан.
- [ ] **B3 maintain** — замер в коридоре и вне → «В коридоре N%» меняется (доля замеров).
- [ ] **B4 правка записи** (✎) — значение меняется → процент/график/вехи пересчитываются **без F5**.
- [ ] **B5 удаление записи** (✕, с подтверждением) → пересчёт без F5.
- [ ] **B6 график** — при ≥2 записях рисуется спарклайн (accumulate — нарастающий, остальные — замеры).
- [ ] **B7 пустая/нечисловая запись** → понятная ошибка, без молчания.

## C. Вехи
- [ ] **C1 добавить веху** (название + порог) → появляется реактивно (○).
- [ ] **C2 авто-зачёркивание** — запиши прогресс через порог → веха ✓ и перечёркнута **без F5**.
- [ ] **C3 порог accumulate** > цели → ошибка; ≤ цели → ок.
- [ ] **C4 удалить** не достигнутую веху (✕).
- [ ] **C5 maintain/rollup** — форма вех скрыта/неприменима, не падает.

## D. Подцели + rollup + навигация
- [ ] **D1 «+ Подцель» на цели БЕЗ записей** → открывается форма, создаётся подцель.
- [ ] **D2 «+ Подцель» на цели С записями** → сразу модалка «Подцели недоступны» (форма НЕ открывается).
- [ ] **D3 rollup** — у родителя % = среднее % подцелей; «N из M подцелей»; своя форма записи скрыта.
- [ ] **D4 переход в подцель** (клик) → экран меняется на подцель (без F5).
- [ ] **D5 «↑ К цели «…»»** — с подцели возврат к родителю (с именем родителя).
- [ ] **D6 глубина** — подцель подцели свыше лимита → ошибка 422 (понятная).

## E. Прогноз (forecast) + срок
- [ ] **E1 нулевой прогресс + срок** → прогноз **молчит** (нет «стоит поднажать»), виден «Осталось N дн.».
- [ ] **E2 есть прогресс + срок** → forecast (в графике / с опережением / стоит поднажать) + «при темпе — к дате».
- [ ] **E3 maintain** — forecast/прогноз-к-дате **отсутствуют** (только коридор + %).
- [ ] **E4 без срока** — подсказка «поставь срок, чтобы видеть прогноз».

## F. Lifecycle
- [ ] **F1 авто-завершение** — accumulate/reach достиг цели → статус «Достигнута» (сам, без F5).
- [ ] **F2 maintain не завершается** никогда.
- [ ] **F3 меню «⋯»** — пауза/снять/архив/восстановить по статусу; пауза замораживает серию/прогноз.
- [ ] **F4 удаление записи у завершённой** — статус остаётся «Достигнута» (по дизайну).

## G. Фокус (·24) + mission-filter (·25)
- [ ] **G1 звезда ☆→★** на карточке → цель уходит в секцию «⭐ В фокусе» (реактивно).
- [ ] **G2 порядок** — несколько в фокусе идут по порядку добавления.
- [ ] **G3 мягкий лимит** — 4-я в фокус → мягкая подсказка «многовато», но **добавляет** (не блок).
- [ ] **G4 убрать из фокуса** (★→☆) → возвращается в «Остальные».
- [ ] **G5 фокус в детали** — меню «⋯» «⭐ В фокус», бейдж «⭐ В фокусе» в шапке.
- [ ] **G6 mission-filter** — accumulate **без** «ради чего откажусь» → звезда в фокус → **отказ** с понятным текстом; заполни через «Изменить» → фокус срабатывает.
- [ ] **G7 reduce/reach/maintain в фокус** — tradeoff НЕ требуется.
- [ ] **G8 пример (стартер) в фокус** → отказ «сначала Добавить себе».
- [ ] **G9 дашборд** — фокусные первыми, с ⭐; примеры исключены.

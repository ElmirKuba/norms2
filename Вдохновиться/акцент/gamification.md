# Gamification

Механики начисления очков, серий, уровней и достижений. Вся агрегация — на бэке. Фронт только отображает.

См. также: [`./domain-model.md#pointevent`](./domain-model.md#pointevent), [`./api-contracts.md#stats--gamification`](./api-contracts.md#stats--gamification).

## Почему работает геймификация

Четыре принципа, по которым игры затягивают — и как каждый реализован в Ascent:

**1. Добровольное участие**
В игре никто не заставляет. Цели, привычки и правила задаёт сам пользователь — не внешние KPI и не чужие шаблоны. "Три победы" (spiritual / physical / mental) — его категории, его формулировки. Ascent не навязывает повестку, он обслуживает твою.

**2. Чёткие цели и микроцели**
Размытая цель ("стать лучше") не работает. `Goal` с `targetValue` и `currentValue` ("Бег: 12.5/77 км") — конкретна и измерима. Ежедневный `Task` — это микроцель, выполнимая за один тап. `WeeklyGoal` — промежуточный горизонт между днём и большой целью.

**3. Мгновенная обратная связь**
Человек, почувствовавший скуку и открывший TikTok, как правило ничего не добивается — потому что привыкает к мгновенному удовольствию без усилия. Ascent конкурирует с этим: отметил задачу → мгновенный toast `+10 XP`, donut обновился по WebSocket, серия выросла. Никаких задержек, никакого "посмотришь результат завтра". Обратная связь — сейчас.

**4. Понятные правила**
Игрок должен точно знать, что происходит и почему. Поэтому `gamification.md` открыт — пользователь видит за что сколько очков, как считается серия, при каком % дня серия не сбрасывается. Никакой "магии за кулисами". Это также означает: правила не меняются без предупреждения.

## Очки

### Источники и значения

| Source | Точки | Условие |
|---|---|---|
| `task` | 10 | Базовое — выполнение задачи из шаблона priority=normal |
| `task` (high priority) | 20 | TaskTemplate с priority=high |
| `task` (low priority) | 5 | TaskTemplate с priority=low |
| `task` (one-off) | 8 | Разовая задача без шаблона |
| `goal_entry` | round(value × 5) | Минимум 5 очков за запись прогресса по цели; масштаб зависит от единицы (для целей в км/часах — иначе шкала может быть слишком высокой; см. секцию "Калибровка") |
| `anti_habit_milestone` | bonus | За каждый milestone-день таймера (3, 7, 14, 30...) — начисляется через BullMQ-job запланированный в момент старта таймера |
| `workout_session` | 50 | За завершённую тренировку |
| `streak_milestone` | bonus | См. таблицу серий |

Значения вынесены в `nest/src/modules/gamification/points.config.ts`. Менять только через PR с обоснованием — баланс важен.

### Идемпотентность

Уникальный ключ `(source, sourceId)` гарантирует, что за один факт не начислится дважды. Это критично:
- Повторный `complete` уже выполненной задачи → no-op.
- Снятие выполнения (`uncomplete`) → существующий `PointEvent` помечается `revokedAt = now()`. При повторном выполнении — создаётся новый `PointEvent` с новым ID (старый остаётся revoked).

### Калибровка очков за прогресс цели

Дефолтная формула `round(value × 5)` хорошо работает для целей в "штуках" (приседания, страницы, дни). Для километров или часов можно поставить меньший множитель. Поэтому в `Goal` (опционально) добавляется `pointsPerUnit` в фазе 7 — пока используется дефолт.

## Серии (Streaks)

### Виды серий

1. **Per-task streak** — для конкретного `TaskTemplate`: число последовательных дней, в которые задача из шаблона была выполнена. День, в который задача не материализована по RRULE, не сбрасывает серию (просто не растит).
2. **Per-anti-habit streak** — непрерывный таймер: `floor((now - currentAttemptStartedAt) / 86_400_000)`. Сбрасывается только нажатием "Рецидив". Не требует ежедневных отметок.
3. **Overall daily streak** — день считается "выполненным", если ≥ X% задач на день выполнены (X настраивается, дефолт = 70%). Серия — последовательность таких дней.

### Алгоритм пересчёта

Базово — on-demand: считается через window function при запросе `/api/me/stats`. Это работает до объёмов в десятки тысяч записей одного пользователя.

В фазе 7 при необходимости (бенчмарк) можно материализовать в `streak_state` таблицу, обновляемую через slack за 5 секунд после каждого изменения.

### Бонусы за серии (`streak_milestone`)

Срабатывает на каждом круглом значении серии:

| Длина серии | Бонус |
|---|---|
| 3 дня | 30 очков |
| 7 дней | 70 очков |
| 14 дней | 150 очков |
| 30 дней | 350 очков |
| 60 дней | 800 очков |
| 90 дней | 1500 очков |
| 180 дней | 4000 очков |
| 365 дней | 10000 очков |

Срабатывает один раз для каждой комбинации `(streakType, streakOwnerId, milestone)`.

### Анти-привычки (timer-модель)

Серия для AntiHabit — непрерывный таймер, не счётчик дней с отметками:

- Серия = `floor((Date.now() - currentAttemptStartedAt) / 86_400_000)` дней — фронт считает в реальном времени.
- **Рецидив** сбрасывает таймер: `currentAttemptStartedAt = now()`, `attemptNumber += 1`.
- Рекорд обновляется при рецидиве если текущая серия > предыдущего рекорда.
- BullMQ-cron для ежедневных проверок не нужен. Milestone-очки начисляются через delayed job: при старте/рестарте таймера планируются jobs на `startedAt + 3d`, `+ 7d`, `+ 14d` и т.д. При рецидиве — pending jobs отменяются.
- Нет понятия "забыл отметить" — либо держишься, либо нажал кнопку.

## Уровни

Формула: `level = floor(sqrt(totalPoints / 100))`.

Примеры:
| Уровень | Нужно очков |
|---|---|
| 1 | 100 |
| 2 | 400 |
| 3 | 900 |
| 5 | 2500 |
| 10 | 10000 |
| 20 | 40000 |

Это даёт мягко возрастающую сложность — первые уровни быстрые, дальше медленнее. На фронте отображается прогресс-бар до следующего уровня.

## Достижения

Стабильный каталог. Каждое достижение имеет `code` (не меняется), название и описание (локализуемые). Каждое выдаётся один раз.

| Code | Триггер |
|---|---|
| `first_task` | Первая выполненная задача |
| `first_goal_completed` | Первая закрытая цель |
| `first_workout` | Первая завершённая тренировка |
| `streak_3_days` | Per-anti-habit или overall streak ≥ 3 |
| `streak_7_days` | Аналогично, ≥ 7 |
| `streak_30_days` | ≥ 30 |
| `streak_90_days` | ≥ 90 |
| `streak_365_days` | ≥ 365 (в одну строку — "год" должен быть ярким событием) |
| `early_bird_7` | 7 дней подряд выполнена задача с тегом `wake_early` |
| `iron_will_30` | 30 дней без срывов на любой anti-habit |
| `complete_week` | Неделя со 100% выполнения по дням |
| `three_pillars` | Активные цели во всех трёх категориях (spiritual, physical, mental) — баланс |
| `hundred_tasks` | 100 выполненных задач суммарно |
| `thousand_tasks` | 1000 |
| `level_5`, `level_10`, `level_25` | Достижение соответствующего уровня |
| `comeback` | Серия восстановилась после сброса до 7+ дней |

Каталог реализован как enum + `AchievementsCatalog` сервис. Расширение — через миграцию (если требуется новая колонка в metadata) и добавление в каталог.

## Доменные события

`@nestjs/event-emitter`. На них подписан `gamification`:

| Event | Источник | Слушает gamification |
|---|---|---|
| `task.completed` | `TasksService.complete` | да — начисляет 10 очков, проверяет first_task, проверяет серии |
| `task.uncompleted` | `TasksService.uncomplete` | да — revoke очков, пересчёт серии |
| `goal_entry.created` | `GoalsService.addEntry` | да — начисляет очки за value |
| `goal.completed` | `GoalsService.addEntry` (когда target достигнут) | да — first_goal_completed |
| `anti_habit_day.set` | `AntiHabitsService.setDay` | да — если `held` — очки и серия |
| `workout_session.completed` | `WorkoutsService.completeSession` | да — first_workout, очки |
| `weekly.summarized` | `WeeklyService` (cron в воскресенье вечером) | да — `complete_week` если 100% |

Слушатели изолированы в `gamification.module` и обновляются независимо от source-модулей.

## Защита от абуза

Чисто self-hosted, abuse — это сам пользователь, обманывающий себя. Минимум защиты, чтобы данные оставались осмысленными:

- За один и тот же `Task` — максимум 1 `PointEvent` (через unique constraint).
- За один и тот же AntiHabit за один день — максимум 1 `PointEvent` (`(source='anti_habit_held', sourceId='<antiHabitId>:<date>')`).
- Для разовых задач — нет защиты от создания миллиона задач за день; это решает сам пользователь. Можно в фазе 9 добавить мягкое предупреждение "ты создал >50 задач сегодня — точно?".

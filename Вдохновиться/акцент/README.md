# docs/ — индекс технической документации

Этот каталог — детализация [`Technical-assignment.md`](../Technical-assignment.md). Каждый файл отвечает за одну область и ссылается на остальные при необходимости.

## Карта

| Файл | Когда читать |
|---|---|
| [`architecture.md`](./architecture.md) | Когда нужно понять общие границы системы, слои и потоки данных |
| [`domain-model.md`](./domain-model.md) | Когда работаешь с сущностями: Goal, Task, AntiHabit и т.д. |
| [`database.md`](./database.md) | Когда пишешь миграцию или сложный запрос |
| [`api-contracts.md`](./api-contracts.md) | Когда добавляешь endpoint или DTO; здесь живёт REST-спецификация |
| [`backend.md`](./backend.md) | Когда работаешь в `./nest/` |
| [`frontend.md`](./frontend.md) | Когда работаешь в `./angular/` |
| [`gamification.md`](./gamification.md) | Когда трогаешь очки, серии, достижения |
| [`ui-ux.md`](./ui-ux.md) | Когда верстаешь экран или меняешь стиль |
| [`deployment.md`](./deployment.md) | Когда трогаешь Docker, переменные окружения, бэкапы, продакшн |
| [`getting-started.md`](./getting-started.md) | Когда нужно поднять проект локально с нуля |

## Принципы документации

1. **Один источник истины на каждую тему.** API-контракты живут только в `api-contracts.md`. Если они дублируются — это баг документации.
2. **Ссылки вместо копий.** Если документу нужны детали из другого — `[название](./other.md#section)`, не повторять.
3. **Изменение поведения = изменение docs в том же коммите.** Без исключений.
4. **Свежесть важнее полноты.** Лучше короткий актуальный документ, чем длинный устаревший.

# ADR-0049: Продуктовые принципы «Акцента» — Внешняя память и Самокомандование

- Статус: принято (2026-06-15, фаза 2 / вычитка доки)
- Контекст: при ревизии «Акцента» сформулированы два ядровых принципа, которые задают
  *смысл* раздела (не механику). Записаны в `README §6` раздела; здесь — каноничное
  «почему» + опора на признанные практики. Дополняют принципы README/ADR-0027.

## Принцип 1 — Внешняя память (разгрузка головы)

Голова — для мышления, не для хранения (**GTD**, David Allen: «mind for ideas, not
for holding them»; **Building a Second Brain**, T. Forte; когнитивная разгрузка;
**эффект Зейгарник** — открытые петли тревожат). Все намерения/планы/контекст живут
в системе — **единый источник истины** (как store на фронте), не в уме. Система не
вываливает всё, а **отдаёт нужный срез в нужный момент** (just-in-time / progressive
disclosure): БД = store, `Recommender`/дашборд = «селектор» над ним.

## Принцип 2 — Самокомандование

Человек в моменте (уставший, в тумане) плохо решает «что делать»; он же в ясном
состоянии — хороший командир. Поэтому: **план задаёшь заранее → система в моменте
директивно ведёт по нему** («Сейчас: …», 1 действие в один тап), масштаб под состояние,
**но никогда не виня за пропуск**. Команда легитимна, потому что **своя** (commitment
device; автономия по теории самодетерминации, SDT) — не ломает мотивацию, а держит.
Самокомандование — активное проявление Внешней памяти.

## Решение

Принять оба как **ядровые принципы раздела** (README §6). Guardrail: директивный тон ≠
вина — тон «интерес, не страх» обязателен (защита от Skinner-box / страха).

## Последствия

- Ориентир для `Recommender` (подфаза 2.7) — отдаёт директивную «команду сейчас».
- Ориентир для **дашборда** (2.10) — «Сейчас/рекомендации» = герой экрана, не пассивный
  список из 20 задач.
- Согласуется с позиционированием (свободно/бесплатно/open-source, без paywall) и
  с anti-burnout/non-Skinner курсом (memory `accent-sustainable-achievement-design`).

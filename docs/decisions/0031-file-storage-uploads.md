# ADR-0031: Работа с файлами и загрузки (платформенно)

- **Статус:** accepted
- **Дата:** 2026-05-31
- **Решает:** Elmir
- **Контекст-теги:** [platform] [files] [phase1] [novaskil]

## Контекст
В фазе 1 загрузка файлов была сознательно отложена (не выбрано хранилище). НоваСкил (фаза 3) требует файлы на диске (контент/обложки/медиа, [ADR-0029](./0029-novaskil-phase3-core.md)). Elmir: «работа с файлами пусть будет если надо; аватарки сможем грузить когда-то». Снимаем двойственность.

## Решение
- **Хранилище файлов — локальный диск** (папка `content/`), NestJS отдаёт статикой (как в НоваСкил). Единый подход для всей платформы; внешнее объектное хранилище (S3 и т.п.) — возможная будущая замена за тем же сервисным портом, без переписывания доменных слоёв.
- **Загрузка файлов разрешена платформенно** (multipart), с лимитами типа/размера (детали в `backend.md`/`deployment.md`). Бэкапы учитывают `content/`.
- **Аватарки ЛК (фаза 1):** **задел сейчас, реализация позже** (после деплоя ЛК):
  - в доке закладываем: поле `accounts.avatar` (nullable, путь к файлу `content/avatars/<id>...`), эндпоинты `POST/DELETE /accounts/me/avatar`;
  - **в MVP фазы 1 НЕ реализуем** — выкатываем ЛК без аватарок, добавим волной. Помечено как отложенное.
  - аватар — изображение (jpg/png/webp), разумный лимит размера; не ПДн (псевдо-аватар, [ADR-0001](./0001-data-minimization-no-pii.md)).
- **Картинки/медиа уроков и обложки курсов** (НоваСкил) — на диске с самого раздела ([ADR-0029](./0029-novaskil-phase3-core.md) N3/N4).

## Альтернативы
- Внешнее хранилище (S3) с фазы 1 — избыточно для self-hosted «для своих». Отложено.
- Совсем без файлов в ЛК — отвергнуто: аватарки нужны «когда-то», лучше заложить задел.

## Последствия
- `database.md`: `accounts.avatar varchar null` (поле есть, UI — позже).
- `api-contracts.md` (фаза 1): эндпоинты аватара помечены «задел, не в MVP».
- `deployment.md`: папка `content/` (включая `avatars/`) — в бэкапах; лимиты upload.
- Снимает «отложенность файлов» из фазы 1 на уровне доки: хранилище выбрано (диск).

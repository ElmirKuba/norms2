# ADR-0016: Формат первичного ключа — строка `uuidv7___unixmillis`

- **Статус:** accepted — **сквозная конвенция проекта.** Применяется ВСЕГДА и ко ВСЕМ таблицам/сущностям, во всех фазах и разделах (ЛК, Акцент, НоваСкил, мессенджеры). Любой новый `id`/FK — в этом формате по умолчанию.
- **Дата:** 2026-05-29
- **Решает:** Elmir (пункт E1)
- **Контекст-теги:** [db] [domain]

## Контекст
Выбирали тип PK. Рассматривали bigint, нативный uuid v7, ULID. Elmir выбрал строковый составной идентификатор как более безопасный и единообразный.

## Решение
**PK всех таблиц — строка** фиксированного формата:

```
<uuid-v7>___<unixtime-ms-13>
пример: 019e7488-0147-7305-9b95-a553f2d00c8e___1780071500548
```

- Левая часть — uuid v7 (36 символов с дефисами).
- Разделитель — `___` (три подчёркивания).
- Правая часть — unix-время в **миллисекундах**, 13 цифр.
- Итоговая длина — 52 символа. Тип колонки — `varchar(52)` (или `char(52)`).
- Все **FK** ссылаются на этот же строковый идентификатор.
- Генерируется в приложении (бэк): сгенерировать uuid v7, добавить `___` и текущее время в мс.

## Обоснование (Elmir)
Единый, самоописательный идентификатор; «так безопаснее» — нет числовой перечислимости, и метка времени явно присутствует в самом id.

## Принятые компромиссы (зафиксировано осознанно)
- uuid v7 уже содержит ms-таймстамп в первых 48 битах, и `created_at` есть на каждой таблице — то есть время хранится с запасом. Принято ради явности/единообразия id.
- Строковый PK (~52 байта) тяжелее нативного uuid (16 байт): индексы и FK во всех таблицах больше, джойны по строке медленнее бинарного. Принято осознанно; масштаб закрытого сообщества это допускает.

## Последствия
- `database.md`: тип `id varchar(52)` PK на всех таблицах; все FK — `varchar(52)`. Валидация формата на уровне приложения.
- **Генерация — util на обоих концах:** функция/метод `generateId()` есть и на бэке, и на фронте (одинаковый формат `uuidv7() + '___' + Date.now()`), чтобы при необходимости сгенерить новый id где угодно. Бэк — авторитетный для персистентности; фронт может генерить (например, оптимистично) тем же утилом. Задокументировать в `backend.md` и `frontend.md`.
- `domain-model.md`: value object `Id` с валидацией формата — чтобы домен не оперировал «голой строкой».
- Закрывает E1 (вместе с конвенцией `created_at`/`updated_at` и суррогатным `id` на всех таблицах).

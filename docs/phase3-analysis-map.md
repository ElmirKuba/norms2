## 0. Что реально в источнике (прочитан весь, 625 стр)

«НоваСкил» по протоколу — **платформа онлайн-курсов (LMS)**:
- **Иерархия контента:** `index.json → Курс → Глава → Урок`. Контент хранится **в JSON-файлах на диске** (не в БД), картинки/обложки/аватары — файлами, NestJS раздаёт `content/` как статику.
- **Прогресс** — в PostgreSQL, фиксируется только на уровне урока; глава/курс «пройдены» вычисляется на фронте.
- **Роли:** `admin` (CRUD контента, управление юзерами) и `user` (чтение + прогресс). Контент правят только админы через редактор **TipTap**.
- **Auth:** email+password, JWT access+refresh, сессии по устройствам (кик устройств).
- **Стек источника:** Angular + NestJS + PostgreSQL + **Drizzle** ORM; **два отдельных репозитория**; id формат `uuidv7_unixtime13digits` (1 подчёркивание).
- Урок: массив контента `[{type: md|video|link|html, src}]`. Курс: level, lang, prerequisites (др. курсы), cover, certificate(bool).
- МВП исключает: канбан, todo.

## 2. Леджер идей (что берём, переформулировано)

- **Каркас контента:** Курс → Глава → Урок; урок = упорядоченный массив блоков `{type, src}` (md/video/link/html — расширяемо). Prerequisites курсов, level, lang, draft/published, version, order, tags.
- **Прогресс** по уроку (отметка пройдено/снято), агрегаты курса/главы — на фронте/сервере. Идемпотентно.
- **Редактор** контента (визуальный + исходник md/html) — TipTap (headless, ок с нашими принципами).
- **Скелетоны** при загрузке; ленивая последовательная загрузка курс→главы→уроки.
- **Сертификат** курса (bool-флаг) — опционально.
- **Версионирование** контента (version++ при правке).

# ADR-0053: Цели — «Фокус» (anti-sprawl) и mission-filter для накопительных целей

- Статус: принято (2026-06-24, фаза 2, подфазы 2.5·24 и 2.5·25, реш. Elmir) — **обратимо**
  (`focus_order` — аддитивная колонка; пороги/строгость — env/копирайт, без миграций данных)
- Контекст: ядро «Целей» (2.5·1…·23) доведено и оттриажено. Чтобы фича была «10/10 по трём
  шляпам», остаются два решения уровня **души продукта**, а не механики: (1) как не дать целям
  расплыться в свалку из 20 штук («1 из 100, фокус, не объём», ADR-0049); (2) как не дать
  накопительным целям превратиться в беговую дорожку «делай больше», против которой создан Акцент
  («тренажёр отказа», ADR-0049). Оба — продуктовые развилки, поэтому за Elmir; зафиксированы здесь.

## Линза 3 шляп

### 1. Бизнес-аналитик
- Без управления приоритетом при 15 целях непонятно, «что сейчас главное» — типичная боль
  трекеров. Конкуренты дают ручной порядок/приоритет.
- Накопительные цели («прочитать 24 книги», «100 км») в массе — это «добавление». Лучшие
  продукты про осознанность сопротивляются голому «делай больше».

### 2. Системный аналитик
- **Фокус — отдельная ось поверх `status`:** «активных может быть много, но в фокусе — единицы».
  Одно поле `goals.focus_order` (integer, null) несёт **обе** роли (combo A+C): `null` = активна,
  но не в фокусе; не-`null` = в фокусе И ранг (для порядка). Без второй колонки.
- Порог фокуса — **мягкий** (combo A, не B): при попытке держать > N в фокусе показываем вопрос
  «что-то отпустишь?», но **не блокируем**. `N` — env `ACCENT_GOAL_FOCUS_SOFT_LIMIT` (default **3**),
  меняется без правок кода (образец — `ACCENT_GOAL_MAX_DEPTH`).
- Ранг при добавлении в фокус = `max(focus_order)+1` (в конец). Ручной drag-порядок — отдельная
  сквозная фича `2.5·27` (общий drag-reorder, server-persist `position`); здесь только ось.
- Mission-filter — **прогрессивная строгость** (combo A+B): мягкая подсказка всегда; жёсткое поле
  «ради чего откажусь» (`tradeoff`) — **условно**, только при заводе accumulate-цели В ФОКУС (цена
  слота фокуса). Связывает ·25 с ·24: трение ровно там, где максимум обязательства.

### 3. Эксперт по помощи людям
- Фокус-лимит — **мягкий**: тренирует выбор (отказ), но не наказывает на ранней стадии (доверие
  важнее в v1; к жёсткому лимиту можно ужесточить позже — обратимо). Жёсткий запрет рискует
  оттолкнуть до того, как человек полюбил продукт.
- Mission-filter: обычная accumulate-цель остаётся **лёгкой** (UI проще ленты, ADR-0049) — только
  мягкий вопрос-рефлексия рядом с «зачем». «Назвать цену» (обязательный `tradeoff`) требуем лишь
  при коммите в фокус, где это уместно.
- Копирайт — просто и без вины (см. правило UI-текстов: по-человечески, механику — в код/доку).

## Решение (итог)

**·24 Фокус (combo ①A+C):**
1. Колонка `goals.focus_order` (integer, **nullable**): `null` = не в фокусе; не-`null` = в фокусе + ранг.
2. **Мягкий** порог `ACCENT_GOAL_FOCUS_SOFT_LIMIT` (env, default **3**): при добавлении сверх порога —
   вопрос «уже N в фокусе — что-то отпустишь?», **без блока** (ответ «всё равно» проходит).
3. Toggle фокуса (`POST/DELETE /accent/goals/:id/focus`); добавление → ранг `max+1`; список сортирует
   секцию «В фокусе» по `focus_order`.
4. Ручной drag-порядок — в `2.5·27` (общий drag-reorder); тут ось/ранг при добавлении.

**·25 Mission-filter (combo ②A+B):**
1. Для `direction='accumulate'` при создании — мягкая подсказка-рефлексия рядом с «зачем» (не блок).
2. Условно-обязательное поле «ради чего откажусь» (`tradeoff`) — **только** при переводе accumulate-цели
   в фокус (цена слота). Для reduce/reach не нужно (они уже «вычитание»/замер).

## Влияние на нарезку
- `2.5·24` — `focus_order` (миграция), env, порт/репо (`setFocus/clearFocus/countFocused/maxFocusOrder`),
  домен `toggleFocus` (возвращает `overLimit`, не бросает), эндпоинт, фронт (секция «В фокусе» + мягкая
  модалка).
- `2.5·25` — `tradeoff` (поле, условная валидация при заводе accumulate в фокус), копирайт-подсказка.

## Обратимость
`focus_order` и `tradeoff` — **аддитивные** колонки. Порог фокуса — env (правка без кода/миграций).
Ужесточить мягкий лимит до жёсткого инварианта позже = доменная проверка + 409, без миграции данных.
Mission-filter — копирайт + одна условная валидация; убрать/усилить тривиально.

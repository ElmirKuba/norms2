# ADR-0006: Правила полей регистрации — alias, login, secret QA

- **Статус:** accepted — **часть про `SecretQA` (вопрос на регистрации) заменена [ADR-0008](./0008-account-recovery-secret-questions.md)** (вопросы в настройках, 1:N). Правила `alias`/`login` и чёрный список тем — в силе.
- **Дата:** 2026-05-29
- **Решает:** Elmir + ToT в [`../archive/decision-map.md`](../archive/decision-map.md) (пункты D5, D6, D7; закрывает B2)
- **Контекст-теги:** [domain] [152fz]

## Контекст
Нужны конкретные правила валидации полей фазы 1, согласованные с режимом минимизации ПДн ([ADR-0001](./0001-data-minimization-no-pii.md)) и раздельной моделью сущностей ([ADR-0005](./0005-user-split-entities.md)).

## Решение

### `alias` (D5) — отображаемое имя, НЕ уникальное
- Длина 2–32. Unicode-буквы (кириллица допустима: «БабаЯга»), цифры, пробелы, дефис.
- Нормализация: trim + схлопывание повторных пробелов. Управляющие символы запрещены.
- Цензуры/фильтра мата в фазе 1 нет (локале-зависимо, дорого) — полагаемся на бан-механику; возможное расширение позже.

### `login` (D6) — уникальный, публичный, для входа
- Маска `^[a-zA-Z0-9_]{3,32}$` (латиница/цифры/`_`).
- **Уникальность регистронезависимая** (`Vasya` == `vasya`): хранить canonical-lowercase для проверки уникальности.
- **Зарезервированные логины** (захардкожены, валидация на бэке): `admin`, `root`, `moderator`, `mod`, `support`, `system`, `normis`, `normisy`, `null`, `me` (список расширяемый) — чтобы не выдавали себя за персонал/служебные адреса.

### `SecretQA` (D7) — гибрид: список + «свой вариант»
- Пользователь выбирает вопрос из **курируемого безопасного списка** ИЛИ вводит **свой**.
- Курируемый список (стартовый, расширяемый): любимая еда; кличка первого питомца; любимый фильм; любимая книга; город, который хочешь посетить; любимая компьютерная игра. — намеренно без тем, намекающих на личность.
- Путь «свой вопрос»: свободный текст **+ предупреждение про ПДн в UI + валидация по чёрному списку тем** на бэке. Чёрный список (ключевые слова, расширяемый): девичья фамилия, фамилия матери, дата рождения, адрес, прописка, номер школы, номер паспорта, ИНН, телефон, номер карты.
- Ответ нормализуется (trim + lowercase + схлопывание пробелов), мин. длина 2, затем **хешируется** (алгоритм — пункт E7).

## Альтернативы
- alias с авто-цензурой — отвергнут для фазы 1 (дорого, ненадёжно).
- login регистрозависимый — отвергнут (двойники-фейки `Vasya`/`vasya`).
- SecretQA только свой вопрос (буква ТЗ) — отвергнут: дырявая валидация PII. Только фикс-список — отвергнут как менее гибкий. Выбран гибрид.

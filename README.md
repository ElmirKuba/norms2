# Нормисы

> Web-платформа с личным кабинетом и набором разделов для своих. Закрытое сообщество по приглашениям, псевдонимная идентификация, приватность по умолчанию.

Это **репозиторий документации и проектирования** проекта «Нормисы». Здесь пока нет кода — здесь то, что важнее: продуманная, непротиворечивая спецификация, по которой код пишется без сюрпризов.

## Что это за проект простыми словами

Платформа, куда попадают **по приглашению** от уже существующего участника. Каждый — под псевдонимом (никаких настоящих имён, телефонов, паспортов). Внутри со временем появятся разделы: трекер целей и привычек, протокол прокачки навыков, мессенджеры. Сейчас делаем фундамент — **личный кабинет**: регистрация, вход, профиль, приглашения, баны, восстановление доступа.

Ключевые принципы:
- 🔒 **Приватность по умолчанию** — собираем минимум данных (логин + псевдоним + пароль), не храним персональные данные.
- 🌳 **Сообщество-дерево** — каждый пригласивший отвечает за приглашённых (может банить своё поддерево).
- 🧩 **Слоистая архитектура** — бизнес-логику можно сохранить при смене БД/ORM.
- 🛠 **Self-hosted** — без облачных зависимостей по умолчанию.

## 🤖 Как написана эта документация

Документация спроектирована **в связке с ИИ** (Claude) по инженерной методике, а не «как получится»:

- Каждое значимое решение принимается через разбор альтернатив и фиксируется отдельной записью — **ADR** (Architecture Decision Record) в [`docs/decisions/`](./docs/decisions/). Сейчас их 18: от модели данных до формата идентификаторов.
- Методика рассуждения (Chain-of-Thought, Tree-of-Thoughts, Self-Consistency) описана в [`docs/methodology.md`](./docs/methodology.md).
- Все открытые и закрытые развилки видны на одной карте — [`docs/decision-map.md`](./docs/decision-map.md).

Цель — документация, по которой реализация идёт без вопросов и багов.

## 🧱 Стек

Angular (SPA, Signals, Tailwind) · NestJS (слоистая архитектура) · PostgreSQL · TypeORM (явные миграции) · Docker · Let's Encrypt.

## 🗺 Фазы

1. **Личный кабинет** (сейчас) — регистрация/вход/профиль/приглашения/баны/восстановление.
2. Раздел «Акцент» — цели, задачи, привычки, геймификация.
3. Раздел «НоваСкил» — протокол прокачки навыков.
4. Облачный мессенджер · 5. E2E-мессенджер · 6. Нативные приложения (Capacitor + Electron).

## Статус

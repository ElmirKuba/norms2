# ADR-0054: Ручная сортировка трекеров перетаскиванием (drag-to-reorder)

- Статус: принято (2026-06-24, фаза 2, подфаза 2.5·27, реш. Elmir) — **обратимо**
  (`position` — аддитивная колонка; UI-механизм заменяем)
- Контекст: трекеры (микро-победы / привычки / цели) сейчас сортируются по `created_at`. Elmir:
  «хочу таскать карточки и менять порядок; и чтобы порядок держался **между устройствами**, а не
  слетал». Это улучшает опыт (шляпа помощи) и должно войти в 2.5 **до деплоя**.

## Решения
1. **Порядок — на сервере** (НЕ `localStorage`, иначе per-device и «вернулось на 3-е место»):
   колонка **`position` (integer, not null)** на `micro_wins` / `habits` / `goals`, **per-account**.
   Листинги: `order by position asc, created_at asc` (детерминированный tie-break).
2. **По умолчанию — как сейчас:** миграция-бэкфилл проставляет `position = row_number() over
   (partition by account_id order by created_at)`. Новые элементы — в конец (`max(position)+1`).
   До первого перетаскивания юзер не замечает разницы.
3. **Эндпоинт реордера:** `PUT /accent/<entity>/reorder`, тело `{ ids: string[] }` (полный
   желаемый порядок видимого списка). Сервер ставит `position = индекс` для **своих** id
   (идемпотентно, без гонок read-modify-write на уровне записи). Чужие/неизвестные id игнорируются.
4. **DnD-механизм — Angular CDK DragDrop** (`@angular/cdk/drag-drop`): **уже в зависимостях** (тянет
   Material, который используется для MatDialog) → новый пакет НЕ добавляется. CDK — это *поведение*
   (touch+mouse, a11y, анимация, auto-scroll), а не Material-UI-кит → правило «Material — только
   MatDialog» (ADR-0025) не нарушено. Свой drag на Pointer Events отвергнут как «больше кода/риска
   без выгоды».
5. **Реактивно, без F5** ([[spa-no-reload-reactivity]]): на `drop` фронт сразу переставляет локально
   (оптимистично) и шлёт `reorder`; на ошибке — откат + модалка.
6. **Сквозная фича:** одинаковый паттерн на 3 трекера (своя колонка/эндпоинт у каждого — общей базы
   нет). Кусок drag-ранга «фокуса» целей (`focus_order`, ADR-0053) можно подключить к тому же DnD позже.

## Обратимость
`position` — аддитивная колонка с бэкфиллом (без потери данных). DnD-механизм (CDK) заменяем на свой
без изменения контракта (`reorder` остаётся `{ ids }`). Откат = игнорировать `position` (сортировать
по `created_at`).

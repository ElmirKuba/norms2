## A. Создание привычек + шаблоны
- [ ] **A1 бинарная** («сделал/не сделал») — название + повторяемость → создаётся в «Шаблонах».
- [ ] **A2 измеримая** (с целью-числом и единицей) → создаётся; на задаче будет поле ввода значения.
- [ ] **A3 лесенка** — стартовый уровень < целевой → создаётся; подпись «сейчас N → цель M».
- [ ] **A4 повторяемость** — ежедневно / по дням недели → текст повторяемости верный.
- [ ] **A5 валидации** — пустое название / цель ≤ 0 у измеримой / стартовый уровень > целевого → понятная ошибка, форма не закрывается.
- [ ] **A6 переключение вида в форме** (бинарная↔измеримая) → поля (единица/цель/лесенка) появляются/прячутся сразу.

## B. Задачи дня (вкладка «Сегодня») — реактивно, без F5
- [ ] **B1 генерация** — у активной привычки на сегодня есть задача; вид совпадает (кнопка «Сделал» / поле + «Отметить»).
- [ ] **B2 выполнить бинарную** («Сделал») → статус «Сделано» сразу; кнопка меняется на «Отменить».
- [ ] **B3 выполнить измеримую** (ввод значения + «Отметить») → засчитано; значение видно.
- [ ] **B4 отмена** («Отменить») → задача снова открыта; **лесенка откатывается** на прежний уровень (P2#2).
- [ ] **B5 перенос** («→ Завтра») → задача помечена «Перенесено», уходит из активных на сегодня (идемпотентно — повтор не ломает).
- [ ] **B6 повторный «Сделал»** по уже выполненной (двойной клик/гонка) → **идемпотентно**, без дублей и без двойного подъёма лесенки.
- [ ] **B7 статусы** — «Сделано» / «Перенесено» / «Пропущено» отображаются корректным лейблом.

## C. Лесенка (never-miss-twice, анти-выгорание)
- [ ] **C1 подъём** — выполняй привычку подряд по правилу → уровень растёт; **flash «лесенка поднялась»** (≈7 сек, закрывается крестиком).
- [ ] **C2 потолок** — на целевом уровне лесенка не лезет выше цели.
- [ ] **C3 понижение** — пропусти по правилу понижения → уровень падает мягко (не в ноль); flash «лесенка опустилась».
- [ ] **C4 откат при «Отменить»** — отмена выполнения возвращает лесенку как было до клика (C1 ↔ B4 согласованы).
- [ ] **C5 пауза раздела** (дашборд «Поставить на паузу») — серии/лесенка/ролловер заморожены; снятие паузы продолжает.

## D. Сортировка перетаскиванием (·27.4, новое в 2.5)
- [ ] **D1** drag ⠿ привычки в «Шаблонах» меняет порядок визуально.
- [ ] **D2 сохранение** — порядок уходит на сервер (`priority`); при сбое — **модалка ошибки + откат**.
- [ ] **D3 cross-device** — порядок держится после F5 и на другом устройстве.
- [ ] **D4 единый порядок** — задачи дня сортируются тем же `priority` (drag привычек влияет и на порядок задач будущих генераций; сегодняшние держат снапшот — приемлемо).
- [ ] **D5 reorder ≠ лесенка** — перетаскивание НЕ трогает уровень/версию (CAS не задет).

## E. Стартовый пак (инертная витрина)
- [ ] **E1 «Получить пак»** → примеры с бейджем «пример»; идут ПОСЛЕ своих, по порядку (M#A3-1-класс — проверить, что не всплывают вверх).
- [ ] **E2 действие на примере** (выполнить/редактировать) → пример «присваивается» себе (бейдж исчезает).
- [ ] **E3 «Очистить примеры»** → удаляются только непринятые; присвоенные остаются.
- [ ] **E4 повтор «Получить пак»** → без дублей (уже существующие названия пропускаются).

## F. Lifecycle + меню «⋯»
- [ ] **F1 меню «⋯»** на привычке — пункты по состоянию (редактировать / деактивировать и т.п.).
- [ ] **F2 редактирование** → изменения видны сразу, без F5.
- [ ] **F3 деактивация** → привычка уходит из активных, задачи на сегодня по ней не генерятся дальше.
- [ ] **F4 удаление/деактивация привычки с историей** → не падает, история задач консистентна.

## G. Ошибки и общее
- [ ] **G1** любое действие при сбое сети/бэка → **видна модалка-ошибка** (drag/complete/uncomplete/postpone — не молчат).
- [ ] **G2** пустой экран «Шаблонов»/«Сегодня» — понятный плейсхолдер с CTA «Получить пак».
- [ ] **G3** seed/clear/complete/uncomplete/postpone/reorder — везде реактивно, без F5.

# UI / UX

Единая дизайн-система. Источник вдохновения — тёмная dashboard-эстетика из исходного шаблона (бежевые акценты на чёрном), скрещённая с современным мобильным fitness/habit UX (Streaks, Habitica, Apple Fitness).

См. также: [`./frontend.md`](./frontend.md).

## Принципы

1. **Тёмная тема — основная** (как в исходном шаблоне). Светлая — опционально, в фазе 9.
2. **Mobile-first.** Дизайн начинается с экрана 375×667. Десктоп — расширение (sidebar, multi-column).
3. **Цифры важнее декора.** Главный элемент дашборда — числа: % выполнено, серия, очки. Они должны читаться с метра.
4. **Минимум кликов.** Отметить задачу — один тап. Записать прогресс цели — один тап + ввод числа.
5. **Обратная связь — мгновенная.** Любое изменение — оптимистичный апдейт UI. Ошибка — откат с тостом.

## Философия тона: интерес, не страх

Ascent должен ощущаться как игра, а не как дневник стыда.

В игре нет страха быть непринятым или неуспешным — есть только интерес. Это и есть причина, по которой игры затягивают, а большинство трекеров — нет. Задача UI — воспроизвести ту же психологию.

**Правило:** каждый текст в интерфейсе должен двигать вперёд, а не давить назад.

| Ситуация | ❌ Страх (нельзя) | ✅ Интерес (нужно) |
|---|---|---|
| Задача не выполнена | "Не выполнено" | "Серия ждёт продолжения" |
| Серия сброшена | "Серия прервана 😔" | "Comeback начинается сегодня" |
| Цель отстаёт | "Отстаёшь от плана" | "12 км / 77 — продолжаем" |
| Пустой дашборд | "Нет данных" | "День начинается — первый шаг?" |
| Антипривычка нарушена | "Срыв" | "Новая попытка" |

**Важное замечание (Teacher269):** интерес появляется не сразу, особенно когда цель далеко. Именно поэтому маленькие победы и мгновенная обратная связь критичны — они создают петли интереса *до* большого результата, пока мотивация ещё не включилась.

Достижения по этой причине должны быть частыми в начале (первая задача, первая неделя, первый уровень) и редкими позже — это имитирует кривую нарастающего интереса.

## Состояния экранов

Каждый feature-экран обязан корректно обрабатывать:
1. **Loading** — skeleton или spinner. Не пустой экран.
2. **Empty** — `<ui-empty-state>` с понятной CTA (создать первую цель / задачу / ...).
3. **Error** — карточка с описанием и кнопкой "Повторить".
4. **Loaded** — основное содержимое.

Эти 4 состояния должны быть проверены в Storybook (если будем делать) или хотя бы visual review.

## Экран Anti-habit (детальный)

Визуально вдохновлён приложениями типа Quitzilla: большой circular progress + живой таймер.

```
┌────────────────────────────────────┐
│  ← НЕ СМОТРЮ ПОРНО            ⋮   │
├────────────────────────────────────┤
│                                    │
│         ╭──────────────╮           │
│         │   🌳          │  Текущая  │
│         │              │  цель     │
│         ╰──────────────╯  1 год    │
│                                    │
│      351 : 04 : 27 : 14            │
│       дн    ч    мин   сек         │
│   (цвет: accent зелёный если >0,   │
│    красный если первые минуты)      │
├────────────────────────────────────┤
│  Попытка 1     [Рецидив]  Рекорд   │
│                            351 дн  │
├────────────────────────────────────┤
│  История            + комментарий  │
│  ┌────────────────────────────┐    │
│  │ + 9 июня 2025 06:00   ×   │    │
│  │   Начало пути.             │    │
│  └────────────────────────────┘    │
└────────────────────────────────────┘
```

**Детали реализации:**
- Таймер: `Date.now() - currentAttemptStartedAt`, `setInterval(1000)`, пересчёт в дни/часы/мин/сек на фронте.
- Circular ring заполнен пропорционально `currentDays / targetDays` (если цель задана).
- Кнопка "Рецидив" — крупная, с подтверждением ("Ты уверен? Серия сбросится.") + поле для заметки.
- История — хронологически, каждая запись показывает дату и заметку.

## Минимум плохого дня

Когда пользователь открывает приложение вечером и серия под угрозой — карточка задачи показывает минимальную версию:

```
┌─────────────────────────────────┐
│ ○  Зарядка                      │
│    Минимум сегодня: 5 мин       │  ← появляется если > 20:00
│    растяжки                     │
└─────────────────────────────────┘
```

Логика отображения: если `minVersion` заполнен у шаблона + задача ещё `pending` + время > 20:00 локального времени → показываем подсказку. Это не отдельный запрос — данные уже в задаче.

## Урок дня как якорь

Продукт должен работать в том числе когда пользователь не в ресурсе — уставший, без мотивации, хочет открыть YouTube. В этот момент он открывает Ascent и видит **фразу которую сам себе написал заранее** — из ресурсного состояния.

Это не мотивационная цитата из интернета. Это твои слова — тебе.

**В UI:**
- При открытии дашборда, если урок дня ещё не записан — показывается вчерашний (или последний существующий) как "якорь" вверху страницы.
- Небольшая карточка, бежевый акцент, без навязчивости.
- Тапнуть → открыть полный журнал уроков.

```
┌────────────────────────────────────┐
│ 💡 "Серия — это не о воле.         │
│     Это о том кто ты есть."        │
│                         — вчера    │
└────────────────────────────────────┘
```

## Геймификация в UI

- **Очки** — короткий toast `+10` сверху при каждом действии. Не более 1 секунды.
- **Серии** — виджет с большим числом и иконкой 🔥 (или огонь lucide-icon `flame`).
- **Достижения** — модалка с конфетти (CSS-анимация), название, описание, кнопка "Здорово". Звук — опциональный, по умолчанию off.
- **Уровень** — в header: `Ур. 14`, прогресс-бар до следующего рядом мелким шрифтом.

## Анимации

- Только purpose-driven, не декоративные.
- Длительность: 150–250ms.
- Easing: `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-expo) для появлений, `ease-in-out` для перемещений.
- Reduce motion respect: `@media (prefers-reduced-motion: reduce)` отключает все нерезкие переходы.

## Доступность

- Контраст текста ≥ 4.5:1 (для мелкого), ≥ 3:1 (для крупного). Пройти `axe-core` без ошибок к фазе 9.
- Все интерактивные элементы достижимы с клавиатуры. Видимый focus-ring (не убирать `outline`).
- Все иконки-кнопки имеют `aria-label`.
- Слайдеры и чекбоксы — нативные элементы или с `role` и keyboard navigation.

## Не делаем

- ❌ Material Design — слишком "корпоративно" для self-hosted habit-tracker'а.
- ❌ Bootstrap — устаревшая визуальная система.
- ❌ Карусели и слайды на дашборде — всё видно сразу.
- ❌ Большие onboarding-туториалы — продукт самообъясним.

# ADR-0048: Доменные уточнения «Акцента» (ревизия R0/R1)

- Статус: принято (2026-06-15, фаза 2 / вычитка доки)
- Контекст: при сквозной вычитке доменной модели «Акцента» (фаза 2, до кода) всплыли
  четыре уточнения, меняющие модель/план относительно первой редакции ADR-0027/0028.
  Фиксируем их каноничным «почему». Детали и статусы — `docs/impl-phase2-plan.md`.

## 1. Слияние `CheckIn` ↔ `DailyMetric`

`CheckIn` (снимок самочувствия → состояние) и `DailyMetric` (метрики для трендов)
дублировали данные (mood/energy/sleep, один-на-день). **Решение: слить.** `CheckIn` —
единый дневной снимок (надмножество полей), `DailyMetric` убрана; тренды 7/30/90
строятся из `CheckIn`. Один источник правды, нет двойного ввода/рассинхрона.

## 2. Иерархия целей — C+ с конфигурируемой глубиной

Нужна декомпозиция «план → подцель», но **без** бесконечного аутлайнера (он сам =
перегрузка, против anti-overwhelm). **Решение:** подцель = та же `Goal` с
`parentGoalId?`. **Глубина — НЕ хардкод, а конфиг** `ACCENT_GOAL_MAX_DEPTH` (env,
zod fail-fast, дефолт `2` = 1 уровень подцелей); инвариант домена читает конфиг →
превышение = `GOAL_MAX_DEPTH_REACHED` (422); фронт берёт лимит с бэка. Дерево — для
организации; ежедневный поток (`Recommender`) работает на листьях, дерево не топит.
Rollup % родителя из подцелей — формула на подфазе 2.4.

## 3. `StateResolver` — упрощённый старт

Полная модель из 6 состояний (survival/recovery/stability/growth/sprint/maintenance)
вычисляется (не хранится). **Решение:** стартуем с упрощённого набора
(`survival|stability|growth` + `paused`); полный список — поздний инкремент **без
миграции** (умнее резолвер + больше правил рекомендаций), т.к. состояние вычисляемое.

## 4. Метод реализации — самодостаточные подфазы

Фаза 2 строится **инкрементально «от простого к сложному»**: каждая подфаза
самодостаточна (работает сама, не ждёт параллельных/будущих фич), последующие
надстраиваются над предыдущими. Порядок выведен из графа зависимостей доменной
модели → дорожная карта `2.0.0→2.11` (`impl-phase2-plan.md`). Геймификация — additive,
поздно (2.8): трекеры полезны и без очков.

## Последствия

- Применено в **R1a** к тексту доки; **R1b**-аудит докрыл контракты (Goal POST +=
  `parentGoalId?`/`attributes?`; эндпоинты справочников).
- Влияет на подфазы: merge → 2.7; иерархия → 2.4 (+`goals.parent_goal_id`, env-var);
  состояния → 2.7; метод — на всю нарезку фазы 2.

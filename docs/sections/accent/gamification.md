# Акцент — геймификация (Фаза 2)

> Очки, серии, достижения, уровни, **алгоритм адаптивной лесенки**. Сущности — [`domain-model.md`](./domain-model.md). Принципы — [README §6](./README.md), [ADR-0027](../../decisions/0027-accent-phase2-core.md).
>
> **Главный принцип:** геймификация **поддерживает, не казинит**. Очки — отражение прогресса, НЕ валюта (нельзя потратить, нет магазина, нет gate привилегий). Вся агрегация — на бэке (серверная истина), фронт отображает. Анти-метрика: не оптимизируем «время в приложении».

## 1. Почему это работает (4 опоры)
1. **Добровольность** — цели/привычки задаёт сам пользователь.
2. **Измеримость** — каждая цель = счётчик; каждая задача = выполнима «за один тап».
3. **Мгновенная обратная связь** — отметил → сразу toast «+N», прогресс обновился.
4. **Понятные правила** — этот файл открыт; за что сколько очков видно, правила не меняются молча.

## 2. Очки (PointEvent)

XP начисляется **только сервером**, идемпотентно ([domain §11](./domain-model.md)). Значения — стартовые наброски, вынести в конфиг (`points.config`), менять только осознанно.

| Источник (`source`) | Очки | Условие |
|---|---|---|
| `task` (normal) | 10 | выполнение задачи priority=normal |
| `task` (high/urgent) | 20 | high/urgent |
| `task` (low) | 5 | low |
| `task` (one-off) | 8 | разовая без шаблона |
| `task_partial` | пропорц. | частичное ≥ minTarget → доля от полного (мин 5) |
| `micro_win` | 5 | микро-победа (дневной лимит на 1 MicroWin) |
| `goal_entry` | round(value × k) | k подбирается по unit (см. §3), мин 5 |
| `milestone` | 50 | достигнута веха цели |
| `goal_completed` | 200 | цель 100% (разово) |
| `anti_habit_milestone` | bonus | по таблице серий (§5) на 3/7/14/30… |
| `streak_milestone` | bonus | по таблице серий (§5) |
| `daily_lesson` | 5 | заполнен урок дня (1/день) |

**Идемпотентность:** partial-unique `(source, sourceId) WHERE revokedAt IS NULL`. Повтор `complete` — no-op. `uncomplete` → `revokedAt=now` (в сумму не входит); повторное выполнение → новый PointEvent.

## 3. Калибровка очков за прогресс цели
`round(value × k)`: для «штук» (отжимания, страницы) k=5; для км/часов k меньше (иначе перекос). Хранить опциональный `pointsPerUnit` на Goal (волной); по умолчанию — дефолт по типу unit.

## 4. Уровни (R11 — РЕШЕНО)

**Формула:** `level = floor(sqrt(totalXP / 100))` (как в Ascent — мягко растущая сложность; первые уровни быстрые).
- L1=100, L2=400, L3=900, L5=2500, L10=10000 XP.
- Уровень — **визуальный статус, без привилегий** (не казино).
- Альтернатива из спека (`100·N^1.5` + множитель серии ×2 при 100д) — **отклонена** для MVP как усложнение; множитель серий риск «pay-to-win ощущения». Можно пересмотреть волной.

## 5. Серии (Streaks) (R8 — РЕШЕНО: on-demand)

**Виды:**
1. **Per-habit** — последовательные дни выполнения привычки (день без материализации по RRULE не сбрасывает).
2. **Per-anti-habit** — таймер `floor((now − startedAt)/сутки)`; сброс только рецидивом ([domain §7](./domain-model.md)).
3. **Overall daily** — день «зачтён», если выполнено ≥ X% задач дня (дефолт 70%); серия — последовательность зачтённых дней.

**Расчёт:** **on-demand** через window function при запросе статистики ([R8] — материализация `streak_state` отложена до доказанной нагрузки). 

**Минимальная победа держит серию:** выполнение `minTarget` (в т.ч. `partial ≥ minTarget`) засчитывает день — защита от «всё или ничего».

**Freeze (заморозка):** 1 заморозка в неделю спасает серию при пропуске; не накапливается (use-it-or-lose-it). **Пауза-режим** (болезнь/отпуск, [domain §3](./domain-model.md)) замораживает все серии без траты freeze.

**Бонусы за серию (`streak_milestone`):**
| дней | бонус |
|---|---|
| 3 | 30 |
| 7 | 70 |
| 14 | 150 |
| 30 | 350 |
| 60 | 800 |
| 90 | 1500 |
| 180 | 4000 |
| 365 | 10000 |

Срабатывает один раз на `(streakType, ownerId, milestone)`.

## 6. Достижения

Стабильный каталог (`code` неизменен, title/desc локализуемы), каждое — один раз. Частые в начале, редкие позже (кривая нарастающего интереса).

| code | триггер |
|---|---|
| `first_step` | первая выполненная задача |
| `first_micro_win` | первая микро-победа |
| `first_goal` | первая закрытая цель |
| `week_strong` | overall-серия ≥ 7 |
| `iron_will_30` | 30 дней без срыва на anti-habit |
| `habit_master` | 5 привычек × 14 дней |
| `comeback` (Феникс) | серия восстановилась до 7+ после сброса |
| `early_bird_7` / `night_owl_7` | 7 дней действий в нужное время суток |
| `milestone_hunter` | 10 достигнутых вех |
| `level_5/10/25` | достижение уровня |
| `hundred_tasks` / `thousand_tasks` | 100 / 1000 задач суммарно |

`comeback` — приоритет для нашего «Васи»: возвращение важнее идеальной серии.

## 7. Алгоритм адаптивной лесенки (`LadderEngine`) — сердце продукта

Цель: вести `currentTarget` от `minTarget` к `goalTarget` без стресса. Для привычки с `ladder.policy = adaptive`:

```
после фиксации дня по привычке:
  performed = doneValue (для binary: 1 если done)
  если performed >= currentTarget:               // выполнил норму
     easyStreak += 1
     если easyStreak >= EASE_THRESHOLD (деф. 3) и currentTarget < goalTarget:
        currentTarget = min(currentTarget + step, goalTarget)
        easyStreak = 0
        событие ladder.raised  (мягкий toast «поднимаем планку: N»)
  иначе если performed >= minTarget:              // минимальная победа
     easyStreak = 0                               // не растим, но СЕРИЯ ЦЕЛА
  иначе:                                          // меньше минимума / срыв
     easyStreak = 0
     если RECOVERY (по UserState) или подряд 2 недобора:
        currentTarget = max(minTarget, currentTarget - step)   // мягкий откат
        событие ladder.lowered («бережём ритм»)
```

- **`step`** по умолчанию: binary — n/a (target=1, растёт через quantitative-вариант); quantitative — +1 или +10% (округление вверх); timed — +30–60 сек.
- В состояниях `survival`/`recovery` планка **не повышается** автоматически; показываются `minVersion`/MicroWin.
- `policy = manual` → пользователь сам меняет `currentTarget`, система только подсказывает тренд.

Пример «1→100 отжиманий»: день1 minTarget=1 ✅ → … 3 лёгких дня → +1 → … срыв → откат к минимуму, серия цела (partial) → снова рост. Так «Вася» доходит до 100 без обнуления мотивации.

## 8. Анти-чит (self-honesty)
Self-hosted, абуз = обман самого себя — минимум защиты ради осмысленности данных:
- 1 PointEvent на `(source, sourceId)` (unique).
- Бэкдейтинг фиксирует факт, но XP не даёт.
- Дневные лимиты на источники (урок 1/день, MicroWin — 1 на конкретный MicroWin/день).
- Все агрегаты — сервер; фронт не считает очки/серии.

## 9. Событийная модель
Модуль геймификации (`GamificationListener`) подписан на доменные события и начисляет награды, не дублируя прогресс:
`task.completed/uncompleted`, `task.partial`, `micro_win.completed`, `goal_entry.created`, `goal.completed`, `milestone.reached`, `anti_habit.held` / `anti_habit.relapsed`, `weekly.summarized`, `ladder.raised`. Слушатели изолированы, обновляются независимо от source-модулей.

## 10. UI-проявления (детали — [`ui-ux.md`](./ui-ux.md))
Toast «+N» (коротко, не крикливо); XP-бар и уровень в хедере; достижение — модалка (`MatDialog`) с CSS-конфетти; серия — виджет 🔥N; подъём лесенки — мягкое уведомление. Звук — по умолчанию off.

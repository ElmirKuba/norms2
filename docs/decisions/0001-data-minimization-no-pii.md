# ADR-0001: Минимизация ПДн — только `login` + `alias` + `password`

- **Статус:** accepted
- **Дата:** 2026-05-29
- **Решает:** Elmir + ресёрч в [`../152fz.md`](../152fz.md)
- **Контекст-теги:** [152fz] [domain] [db]

## Контекст
Платформа размещается вне РФ и не хочет получать статус «оператора персональных данных» по 152-ФЗ (локализация, уведомление РКН, штрафы по 13.11 КоАП). Любое обязательное поле, позволяющее прямо или косвенно идентифицировать физлицо, переводит проект в режим оператора.

## Решение
В обязательных полях регистрации — только `login`, `alias` (псевдоним), `password`, а также `secret_question`/`secret_answer` для восстановления. Никаких ФИО, телефонов, паспортов, дат рождения и **реальных email**. Опциональный email — только отдельной фичей, с явным согласием и целью (отложено за пределы Фазы 1).

## Альтернативы
- **Сбор email при регистрации** (для сброса пароля и уведомлений) — отвергнут: делает нас ПДн-оператором, тянет локализацию в РФ.
- **Верифицированный профиль (имя/телефон)** — отвергнут как несовместимый с псевдонимной моделью и юрисдикционной стратегией.

## Последствия
- Восстановление пароля — без email (см. отдельный ADR по C4, когда решим).
- `ban_reason` / `invite_reason` — свободный текст с UI-предупреждением «не указывайте реальные данные».
- Секретный вопрос — с захардкоженным чёрным списком тем (девичья фамилия, адрес, дата рождения мамы и т.п.).
- IP/User-Agent — изолированная таблица логов, TTL 60 дней, хэш с дневной солью, без FK на `users`.

# ADR-0008: Восстановление доступа — секретные вопросы в настройках, 1:N, K-of-N

- **Статус:** accepted
- **Дата:** 2026-05-29
- **Решает:** Elmir + ToT в [`../archive/decision-map.md`](../archive/decision-map.md) (пункт D7)
- **Контекст-теги:** [domain] [db] [152fz]
- **Заменяет:** часть про `SecretQA` в [ADR-0005](./0005-user-split-entities.md) (кардинальность 1:1) и [ADR-0006](./0006-registration-field-rules.md) (вопрос на регистрации). Чёрный список тем из ADR-0006 — остаётся в силе.

## Контекст
Изначально секретный вопрос задавался при регистрации (1:1). Elmir уточнил: вопросы настраиваются **после** регистрации в настройках аккаунта, их **много** (1:N), и пользователь сам решает, на сколько отвечать при восстановлении.

## Решение

**Регистрация больше НЕ собирает секретные вопросы.** Поля регистрации: `login`, `alias`, `password` (+ `invite_code` при `FREE_REGISTRATION=false`). См. [ADR-0001](./0001-data-minimization-no-pii.md).

**Настройка восстановления** — отдельная фича для аутентифицированного аккаунта:
- Таблица `secret_qa` **1:N** к `users`: каждая запись — `(id, user_id, question, answer_hash, created_at)`.
- Вопрос — гибрид: из безопасного списка ИЛИ свой (с предупреждением про ПДн + валидацией по чёрному списку тем, ADR-0006).
- На аккаунте — `recovery_required_count` (K): сколько вопросов нужно будет ответить при восстановлении. Юзер задаёт сам, `1 ≤ K ≤ N` (N — число настроенных вопросов).

**Флоу восстановления:**
1. Вводит `login`.
2. Бэк выбирает **K случайных** из его вопросов и отдаёт их тексты.
3. Юзер вводит ответы (плейнтекст по TLS — [ADR-0009](./0009-server-side-hashing.md)), бэк хеширует и сверяет **все K**.
4. Все совпали → задаёт новый пароль.

**Риск без настройки:** не настроил вопросы и забыл пароль → аккаунт невосстановим (email нет). Сознательная цена псевдонимности. Митигатор — **баннер-напоминание в настройках**: «настройте восстановление, иначе потеряете доступ».

## Альтернативы
- Фиксированное K (например, всегда все или всегда 1) — менее гибко. Отвергнут в пользу выбираемого K.
- Вопрос на регистрации (буква ТЗ) — отвергнут: усложняет регистрацию, восстановление логичнее в настройках.

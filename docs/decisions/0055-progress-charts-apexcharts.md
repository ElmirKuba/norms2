# ADR-0055: Графики прогресса — ApexCharts (за своим wrapper-компонентом)

- Статус: принято (2026-06-25, фаза 2, подфаза 2.5·28, реш. Elmir) — **обратимо** (движок изолирован
  за своим компонентом-обёрткой; заменяем без касания вызывающего кода).
- Контекст: график прогресса цели — ручной SVG-спарклайн (`goal-detail`, ADR-0052/2.5·17; шкалы
  добавлены в ·28 G#B1-4). Лёгкий, но без полноценных осей/делений/тултипов/зума. Elmir: «хочу
  графики со шкалами и т.д.» — и сейчас (деталь цели), и под будущие дашборды/кокпит «Сейчас» (2.11).
  Развилка задевает lean-этос проекта (CLAUDE.md: «без раздутых UI-китов»).

## Решения
1. **Движок — ApexCharts** (`apexcharts`) + официальная Angular-обёртка **`ng-apexcharts`**
   (standalone-совместима). Лицензия **MIT**, **self-host** (бандлится в наш билд, без CDN/облака —
   совместимо с «self-hosted без облачных зависимостей»).
2. **Рассмотренные альтернативы (ToT):** свой SVG (0 зависимостей, но тултипы/зум писать самим);
   **uPlot** (~22 КБ gzip, рекомендовался как баланс lean+фичи); **Chart.js** (~50 КБ). **Выбран
   ApexCharts** (реш. Elmir) ради готовых из коробки осей/шкал/тултипов/зума/легенды и зрелой
   Angular-поддержки — **осознанно приняв вес** (~140 КБ gzip + обёртка). Это **исключение** к «без
   раздутых UI-китов» — точечное, ради ценности визуализации; правило «Material — только MatDialog»
   (ADR-0025) не задето (ApexCharts — charting-движок, не UI-кит).
3. **Изоляция за СВОИМ компонентом-обёрткой** (напр. `app-chart` в `shared/ui`): весь остальной код
   знает только наш контракт (данные ряда + тип → график), а НЕ ApexCharts напрямую. Это сохраняет
   **заменяемость движка** (тот же принцип чистых границ, что в бэке): сменить ApexCharts на uPlot/свой
   SVG можно без касания фич-кода. Импорт `apexcharts`/`ng-apexcharts` — **только** внутри обёртки.
4. **Вес не бьёт по initial bundle:** график только на lazy-загруженных экранах Акцента (раздел и так
   lazy-loaded) → ApexCharts уходит в lazy-чанк, не в initial. При нужде — динамический импорт внутри
   обёртки.
5. **Реактивно, наши токены:** график принимает данные из сигналов (без F5, [[spa-no-reload-reactivity]]),
   цвета/шрифты — через CSS-переменные нашей темы (а не дефолтную тему ApexCharts), чтобы не разъезжался
   с дизайн-языком (ADR-0025).

## Обратимость
Движок спрятан за `app-chart` → замена ApexCharts на другой (uPlot/свой SVG) меняет только обёртку,
вызывающий код (деталь цели, будущие дашборды) не трогается. Полный откат = вернуть SVG-спарклайн и
удалить зависимости.

## Последствия
- (+) Полноценные графики из коробки: оси/шкалы/деления/тултипы/зум/легенда; меньше своего кода на фичах.
- (+) Зрелая Angular-поддержка (`ng-apexcharts`), хорошая дока.
- (−) Вес бандла (~140 КБ gzip) — митигируется lazy-чанком Акцента.
- (−) Новая внешняя зависимость на сопровождение (обновления, типы).
- Интеграция = build-system изменение (package.json/lockfile + пересборка dev-контейнера) → отдельная
  задача/коммит (front), не в потоке клик-регресса ·28.

# Раздел «Акцент» — Фаза 2

> Личная ОС достижений внутри Нормисов: цели, привычки, микро-победы и геймификация, которые ведут человека от «1 отжимания» к устойчивому росту. Для **аутентифицированных** пользователей, на общем ЛК фазы 1.
>
> Характер и ключевые решения — [ADR-0027](../../decisions/0027-accent-phase2-core.md), [ADR-0028](../../decisions/0028-accent-timezone-and-domains.md) (timezone+домены), [ADR-0047](../../decisions/0047-accent-naming-convention.md) (имена), [ADR-0048](../../decisions/0048-accent-domain-refinements.md) (доменные уточнения), [ADR-0049](../../decisions/0049-accent-product-principles.md) (принципы), [ADR-0050](../../decisions/0050-accent-module-architecture.md) (архитектура модуля). История переваривания источников — в git (рабочая карта своё отработала и удалена). Детали — в соседних файлах этой папки. Реализация — [`../../impl-phase2-plan.md`](../../impl-phase2-plan.md).

## 1. Что это и для кого

«Акцент» — раздел платформы для постановки целей и формирования привычек, построенный на двух идеях:
1. **Сверх-низкий порог входа** (Tiny Habits / BJ Fogg): 1 отжимание = засчитанная тренировка. Импульс важнее объёма.
2. **Антихрупкость** (SuperBetter): срыв — это данные, а не провал; система помогает вернуться, а не наказывает.

North star: **даже человек в тяжёлом состоянии, делая смешно-маленькие шаги, забирает контроль над жизнью** — и дальше растёт до мастерства. Бесплатно, opensource, «для своих».

**Позиционирование одной фразой:** «Акцент учит **отказываться от лишнего ради важного**» — это тренажёр **вычитания** (власть отказа = свобода), не добавления. Миссия — **расширить «бутылочное горло»** (чтобы прорывался не 1 из 100), растить людей, которые ценят/приумножают/**передают дальше**. См. [ADR-0049](../../decisions/0049-accent-product-principles.md) (Принцип 3 + Миссия).

Аудитория сознательно включает людей в плохом состоянии (депрессия, выгорание, тревога, ADHD, хроническая боль) — отсюда accessibility-first и recovery-режимы (см. §6).

## 2. Привязка к фундаменту (фаза 1)

«Акцент» — **раздел, не отдельный продукт**. Использует общий ЛК:
- идентичность — `accounts` ([фаза 1](../../domain-model.md)); **отдельного User не заводим**, все сущности раздела вешаются на `account_id`;
- идентификаторы — формат `uuidv7___unixmillis` ([ADR-0016](../../decisions/0016-primary-key-format.md));
- backend — 5-слойный Nest (controllers/use-cases/domain-services/adapters/repositories), [архитектура](../../architecture.md), [ADR-0030](../../decisions/0030-stack-revision-drizzle-5layer-npm.md);
- API — `/api/v1`, конверт ошибок, zod-DTO ([ADR-0020](../../decisions/0020-api-conventions.md));
- frontend — Angular standalone + Signals + **чистый SCSS** (без Tailwind), модалки через `MatDialog` ([ADR-0025](../../decisions/0025-ui-ux-design-language.md), [ADR-0026](../../decisions/0026-modal-system.md));
- «Акцент» — отдельный модуль на бэке (`modules/accent/...` или набор под-модулей) и lazy-loaded feature на фронте.

## 3. Состав раздела (каркас — гибрид, [ADR-0027](../../decisions/0027-accent-phase2-core.md))

База (понятно/измеримо, из вдохновения Ascent) + усиления SuperBetter:

| Сущность | Кратко |
|---|---|
| **Goal** (цель) | большая измеримая цель: `whyItMatters`, unit, target/current, deadline?, `fallbackVersion` (версия на плохой день), вехи (Milestones), подцели (`parentGoalId`, глубина из конфига) |
| **GoalEntry** | запись прогресса по цели (дата, значение, заметка) |
| **Habit / TaskTemplate → Task** | повторяемые задачи (RRULE) + материализация на день; разовые задачи (one-off) |
| **MicroWin** | «микро-победа» 10 сек–5 мин — техническая форма «1 отжимание = победа» |
| **AntiHabit** | таймер «держусь N дней», рецидив сбрасывает, рекорд |
| **Obstacle** | препятствие как сущность (триггер/симптомы/контрмеры) |
| **Supporter** | союзник поддержки (через круг доверия / инвайт-дерево) |
| **Identity** | игровая личность, развитая из `alias` ЛК |
| **CheckIn** | единый дневной снимок (mood/energy/sleep/… → выводит UserState; тренды 7/30/90) |
| **PointEvent / Achievement / Streak** | геймификация (идемпотентно, на бэке) |
| **WeeklyGoal / DailyLesson** | недельный фокус, «урок-якорь» |

Полные поля и инварианты — [`domain-model.md`](./domain-model.md).

## 4. Сердце продукта — адаптивная лесенка

- У каждой привычки/цели есть **минимальная победа**. Выполнил минимум → засчитано, **серия сохраняется** (нет «всё или ничего»).
- Система **предлагает следующий шаг**: легко несколько дней → чуть больше; тяжело/срыв → держит или откатывает к минимуму.
- **Частичное выполнение** засчитывается; типы привычек: бинарная / количественная / временная.

Правила лесенки и очков — [`gamification.md`](./gamification.md).

## 5. State-aware (адаптация под состояние)

Полная модель состояний: **survival / recovery / stability / growth / sprint / maintenance**. CheckIn → состояние → рекомендации (1–3 микро-действия). Автоснижение нагрузки при ухудшении, recovery-режим, пауза-режим (болезнь/отпуск) замораживает серии. **Срыв = данные, не провал.**

## 6. Принципы раздела

- **Внешняя память — разгрузка головы (ядро):** голова — для мышления, не для хранения (GTD: «mind for ideas, not for holding them»). Все намерения, планы и контекст живут в системе — **единый источник истины** (как store на фронте), не в уме. Система не вываливает всё разом, а **отдаёт нужный срез в нужный момент** (just-in-time / progressive disclosure): `Recommender` и дашборд — это «селектор» над хранилищем. Так закрываются открытые петли (эффект Зейгарник), падает когнитивная нагрузка (критично при выгорании/ADHD), энергия остаётся на действие, а не на удержание списков в голове. Активное проявление — **Самокомандование** ниже.
- **Самокомандование (ядро):** пользователь задаёт план в ясном состоянии; система в моменте **директивно ведёт** его по этому плану («Сейчас: …», 1 действие в один тап), адаптируя масштаб под состояние, но **никогда не виня за пропуск**. Команда легитимна, потому что **своя** (твоё собранное «я» командует уставшему «я» — commitment device, автономия по теории самодетерминации). Дашборд — не пассивный список из 20 задач, а ясная команда «делай вот это». Влияет на `Recommender` и дашборд.
- **Свободно и бесплатно:** open-source, без paywall и «премиум-мотивации» (в отличие от SuperBetter-ставшего-бизнесом и платных трекеров). Помогаем взять контроль, а не доим за это.
- Сначала безопасность → мотивация; микро-действие → амбиция; восстановление → масштаб; доверие → данные.
- Геймификация **поддерживает, не казинит**: очки — не валюта, нет магазина, нет gate привилегий.
- **Анти-метрика:** не оптимизируем «время в приложении». Цель — реальные изменения в жизни.
- **Тон «интерес, не страх»**: каждый текст двигает вперёд (срыв → «Comeback начинается сегодня»).
- **Accessibility-first** с самого начала: клавиатура, контраст, screen-reader, простой язык, низкая когнитивная нагрузка.
- ПДн не собираем (наследуем [152-ФЗ-режим](../../decisions/0001-data-minimization-no-pii.md)); `reason`/заметки — с предупреждением.

## 7. Объём: MVP и волны

**MVP (подфазы `2.0.0`–`2.12`)** — широкий ([ADR-0027](../../decisions/0027-accent-phase2-core.md) R7): Goal(+why/fallback/вехи)+GoalEntry, Habit/Task (ролловер RRULE)+разовые, AntiHabit, **микро-победы + адаптивная лесенка**, WeeklyGoal, DailyLesson, геймификация (XP/серии/достижения), **модель состояний + CheckIn** (CheckIn — единый дневной снимок, тренды из него), Obstacle, Dashboard-агрегатор. Порядок и детальная разбивка — `docs/impl-phase2-plan.md`.

**Поздние волны (миноры `2.13+`, тоже фаза 2):** Workout (тяжёлый модуль), круг доверия/соц (Supporter-взаимодействие, совместные челленджи), AI-помощник (декомпозиция целей, перепаковка срывов — с границами безопасности), доп-модули (шаблоны целей/привычек, квесты, сезоны, ритуалы, экспорт данных), полный onboarding.

> Реализационная заметка: MVP широкий — при кодинге резать на под-волны 2.0/2.1; в **доке** описываем всё (принцип «ничего не упустить, потом шлифовать»).

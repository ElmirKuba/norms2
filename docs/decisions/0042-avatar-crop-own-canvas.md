# ADR-0042: Кроп аватара — свой canvas, без crop-библиотеки

- Статус: принято (2026-06-08)
- Контекст: фаза 1, F3.3. Бэк принимает **уже нарезанное** квадратное изображение
  (ADR-0032/0040), значит кроп — целиком на фронте. Нужен выбор инструмента.

## Развилка (ToT)

| Ветвь | Bundle | Зависимости | UX | Риск |
|---|---|---|---|---|
| **A. Свой canvas-кроп** | +0 | нет | полный контроль, наш SCSS | сами пишем pointer/zoom-математику |
| B. Лёгкая crop-либа (ngx-image-cropper и пр.) | +~30–50 КБ | +1 npm | готовые жесты из коробки | чужие стили, обновления, совместимость с Ng21 |

## Решение

**Ветвь A — свой canvas-кроп** (реш. Elmir 2026-06-08).

Причины (в порядке веса):
1. **Без раздутости** — прямое следование принципам проекта (свои компоненты,
   чистый SCSS, без CSS-китов/лишних фреймворков; см. CLAUDE.md «не любит»).
2. **Ноль внешних зависимостей** — нечего обновлять/ломаться при апгрейде Angular,
   нет чужих стилей в нашем дизайн-языке.
3. Кроп простой (квадрат под круглый аватар: pan + zoom) — объём своего кода
   умеренный (~200 строк) и полностью под нашим контролем.

## Реализация

- `AvatarCropModalComponent` (MatDialog, Способ B — медиум-модалка): квадратный
  viewport, изображение рисуется на `<canvas>`; pan — pointer-events с захватом,
  zoom — `<input type=range>` (центр сохраняется). Клэмп: изображение всегда
  покрывает viewport (cover-scale × zoom, смещение в допустимых границах).
- Вывод: `canvas.toBlob('image/jpeg', 0.9)` в квадрат OUTPUT×OUTPUT (512), белый
  фон под прозрачность → `File('avatar.jpg')`.
- Открывается через `AvatarCropService` (инжектит `MatDialog`) → `Promise<Blob|null>`,
  чтобы компоненты не дёргали `MatDialog` напрямую (ADR-0026).
- Отправка/удаление — через уже готовые `AccountApiService.uploadAvatar/removeAvatar`
  (F3.2). Бэк валидирует тип (magic-bytes) и размер повторно.

## Последствия

- Полный контроль над UX и стилем; нулевой вес.
- Если позже понадобится сложный кроп (поворот, нестандартные пропорции) —
  пересмотреть (новый ADR), но для круглого аватара ветви A достаточно.

# docs/ — карта документации

Этот каталог — детализация [`../Technical-assignment.md`](../Technical-assignment.md). Каждый файл отвечает **за одну доменную область** и является **единственным источником истины** по ней; остальные на него ссылаются, а не дублируют.

---

## 1. Уровни документации (кто кому главный)

```
┌─ КОРЕНЬ (управляющие файлы) ───────────────────────────────────────────┐
│                                                                        │
│  CLAUDE.md ◄── грузится ВСЕГДА. Тонкий роутер: «как работаем» + карта.  │
│     │           Не содержит деталей — только правила и ссылки.         │
│     ├──► Technical-assignment.md   «ЧТО делаем» — продуктовый scope,    │
│     │                               единый источник нумерации фаз.     │
│     ├──► todo.md                    «ГДЕ мы» — план работ по фазам.     │
│     └──► docs/ (этот каталог)       «КАК устроено» — детали по областям.│
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
        │
        ├─ СКВОЗНЫЕ (процесс и принципы, нужны на любой фазе)
        │     methodology.md · decisions/ · 152fz.md · archive/decision-map.md
        │
        ├─ ИНЖЕНЕРНЫЕ (фаза 1, спек для реализации)
        │     architecture · domain-model · database · api-contracts
        │     · backend · frontend · ui-ux · deployment · getting-started
        │
        └─ РАЗДЕЛЫ (позже)  sections/accent.md (ф.2) · sections/novaskil.md (ф.3) · …
```

**Правило загрузки** (из CLAUDE.md): не читать весь `docs/` сразу. Брать файл под текущую задачу. CLAUDE.md и `Technical-assignment.md` — единственное, что читается всегда в начале.

## 3. Граф связей между инженерными доками

Стрелка `A ──► B` = «A питает B»: решения из A — вход для B. Менять надо по направлению стрелок, иначе ниже по потоку рассинхрон.

```
                  152fz.md ───────────────┐ (ограничивает всё: что нельзя хранить)
                       │                   │
   archive/decision-map.md ──► (каждое решение → ADR в decisions/) ──► питает доки ниже
                       │
   architecture.md ──► domain-model.md ──► database.md ──► api-contracts.md
        │                   │                                   │
        │                   └──────────────┐                    │
        ▼                                  ▼                    ▼
     backend.md  ◄──────────────────── (домен+схема+контракт)   │
        │                                                       │
        └──────────────► api-contracts.md ◄────────────────────┘
                                  │
                                  ▼
                             frontend.md ──► ui-ux.md
                                  │
   deployment.md (стоит сбоку: ENV/инфра) ──┐
                                            ▼
                              getting-started.md (потребляет всё: как поднять)
```

Ключевые зависимости словами:
- **`152fz.md` ограничивает всё** — ни одна сущность/поле/лог не появляется в обход него.
- **`domain-model.md` → `database.md` → `api-contracts.md`** — главная цепочка фазы 1: домен задаёт сущности, БД их материализует, API их публикует.
- **`architecture.md`** задаёт рамку (слои, токены, ошибки), внутри которой живут backend/frontend.
- **`api-contracts.md`** — стык backend↔frontend; правится один раз, читается обоими.
- **`deployment.md`** питается ENV-решениями из `architecture.md`, но в остальном автономен.
- **`getting-started.md`** — потребитель: ничего не решает, только описывает запуск из всего выше.

## 4. Принципы документации

1. **Один источник истины на каждую тему.** Тема живёт в своём файле (см. §2). Дублируется — это баг документации.
2. **Ссылки вместо копий.** Нужны детали из другого дока — `[название](./other.md#section)`, не повторять.
3. **Изменение поведения = изменение docs в том же коммите.** Без исключений.
4. **Свежесть важнее полноты.** Короткий актуальный документ лучше длинного устаревшего.
5. **Решение → ADR.** Развилка, как только закрыта, фиксируется в `decisions/` и больше не «живёт в голове». (Карта развилок фазы 0 — в `archive/decision-map.md`.)

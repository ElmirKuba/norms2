## Как этим пользоваться

**Только как источник идей.** Никогда — как источник истины, кода или формулировок.

В Нормисах эти проекты переосмыслены как **разделы внутри платформы**, а не отдельные приложения. Они используют общий ЛК, общую регистрацию по приглашениям, общую псевдонимную идентификацию из 152-ФЗ-режима. Их внутренние схемы, контракты, миграции, формулировки документации **не подходят напрямую** — даже если выглядят похоже.

## Правила (жёсткие)

❌ **Нельзя** копировать оттуда:
- Схемы таблиц БД и SQL-миграции
- DTO, API-контракты, OpenAPI/Swagger-описания
- Формулировки из их `claude.md`, `Technical-assignment.md`, `README.md`
- Структуру их `docs/` (у нас будет своя)
- Их `todo.md`-пункты как наш план работ
- Их код, если бы он там был

✅ **Можно** оттуда брать:
- Идеи продуктовых фич («очки за выполнение задачи», «серии», «достижения»)
- Описания доменных понятий как материал для **переформулирования** своими словами
- Примеры UX-сценариев — для адаптации, не для копирования
- Архитектурные подходы — как **один из вариантов**, не как «правильный»

## Если возник соблазн скопировать

Остановись. Открой [`../CLAUDE.md`](../CLAUDE.md) пункт TL;DR #5. Если всё ещё кажется, что копировать правильно — **спроси у Elmir-а**, а не делай молча.

  - `новаскил/` — при старте Фазы 3
- **Сейчас (Фаза 0 / Фаза 1)** — не читать. ЛК у нас свой, с нуля.

## Почему всё так строго

Если документация Нормисов начнёт цитировать вдохновители — мы получим:

1. Архитектурный долг: чужие решения, принятые под другой scope.
2. Семантический долг: формулировки, не совпадающие с нашей моделью.
3. Юридический долг: вдохновители могли допускать сбор email/имени; у нас [`docs/152fz.md`](../docs/152fz.md) этого не допускает.

Поэтому правило: **пишем с нуля. Из этой папки — только идеи**.

# ADR-0028: Акцент — timezone в accounts (R6) и сферы+атрибуты целей (R10)

- **Статус:** accepted
- **Дата:** 2026-05-30
- **Решает:** Elmir (развилки R6, R10 из [`../archive/phase2-analysis-map.md`](../archive/phase2-analysis-map.md))
- **Контекст-теги:** [accent] [phase2] [domain] [db]

## R6 — timezone хранится в `accounts` (платформенно)
Ролловер задач и серии Акцента считаются по **локальной полуночи** пользователя → нужен IANA timezone. Решение: **добавить `timezone` платформенным полем в таблицу `accounts`** (фаза 1), а не в настройки раздела.

- Обоснование: TZ — свойство человека, не раздела; единый источник для всех будущих разделов (НоваСкил, мессенджер). Дублирования нет.
- Поле: `accounts.timezone varchar(64) NOT NULL DEFAULT 'UTC'` (IANA, напр. `Europe/Moscow`); проставляется из браузера при регистрации/первом входе, меняется в настройках профиля ЛК.
- Это **аддитивная** миграция, ничего не ломает; фаза 1 ещё не в проде. **Влияет на доку фазы 1:** обновить `database.md` (таблица `accounts`), `domain-model.md` (сущность Account), api-contracts (профиль отдаёт/принимает `timezone`).
- В Акценте `accent_settings.timezone` НЕ заводим; берём из `accounts`. (корректирует §0 в [`../sections/accent/api-contracts.md`](../sections/accent/api-contracts.md))

## R10 — у цели и сфера, и атрибуты (оба)
Цель привязана к **сфере жизни** (факт) И прокачивает **RPG-атрибуты** (игра):

- **`domainKey`** (как было): health/sleep/sport/work/money/relationships/learning/home/creativity/purpose — фактическая область.
- **Атрибуты** (новое): Сила / Дисциплина / Интеллект / Дух / Социальность / Здоровье (набор настраиваемый/расширяемый). Действие/цель начисляет прогресс одному-нескольким атрибутам → **«паучья диаграмма» баланса** (видны перекосы) + «прокачка персонажа» (связь с Secret Identity).
- Маппинг свободный: у цели/привычки — `domainKey?` + `attributes[]` (0..N). Если атрибут не указан — можно вывести дефолт из domain (напр. sport→Здоровье/Сила), но это подсказка, не жёсткое правило.
- XP действия может дробиться по атрибутам для диаграммы; общий уровень — как в [gamification](../sections/accent/gamification.md) (sqrt от totalXP).

## Альтернативы
- R6 в `accent_settings` — самодостаточно, но дублирование TZ для других разделов. Отвергнут.
- R10 только сферы / только атрибуты — отвергнуты: сферы практичны, атрибуты геймовы; вместе сильнее (факт + мотивация), ценой небольшого усложнения модели.

## Последствия
- **Фаза 1 (правка доки + миграция при реализации):** `accounts.timezone`.
- **Акцент domain-model:** добавить набор `Attribute` (справочник) + `attributes[]` на Goal/Habit/PowerUp; убрать timezone из accent_settings; паучья диаграмма — в ui-ux/stats.
- **Акцент ui-ux:** виджет баланса атрибутов (radar/паучья диаграмма) на дашборде/stats.
- **Акцент api:** `/accent/settings` без timezone (берётся из профиля ЛК); в stats — разбивка XP по атрибутам.
- Реализация диаграммы атрибутов может уйти в волну 2.1, но модель закладываем сразу.

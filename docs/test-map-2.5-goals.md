## A. Создание целей — 4 рода
- [ ] **A1 accumulate** — «Накопить», единица, цель >0 → создаётся; бейдж «Накопить».
- [ ] **A2 reach** — «Достичь уровня», цель ≠ старт → создаётся; есть поле «Старт».
- [ ] **A3 reduce** — «Снизить», цель ниже старта → создаётся.
- [ ] **A4 maintain** — «Удерживать», **нижняя < верхняя** → создаётся; бейдж «Удерживать».
- [ ] **A5 валидации:** accumulate цель=0 → ошибка; reach цель=старт → ошибка; maintain без нижней / ниж≥верх → ошибка (понятный текст, форма не закрывается).
- [x] **A6 переключение рода в форме** — выбираешь reach → появляется «Старт»; maintain → «Нижняя/Верхняя граница»; accumulate → подсказка-миссия + «ради чего откажусь». Поля реагируют сразу. _(·28: проверено браузер-автоматизацией — все 4 рода переключают поля на лету.)_

## B. Запись прогресса + история (реактивно, без F5)
> Кейсы расписаны на эталонных целях (создаются в стенде): A1 «Прочитать 30 книг» (accumulate, 9/30),
> A2 «Подтянуться 15 раз» (reach, старт 3, цель 15), A3 «Снизить вес до 75» (reduce, старт 90), A4 «Сон 7–9 ч»
> (maintain, коридор 7–9). Везде проверяем реактивность **без F5** ([[spa-no-reload-reactivity]]).

- [x] **B1 accumulate** — на A1 запиши «+N» → сумма/процент растут сразу; запись появилась в «История».
- [ ] **B2 reach/reduce — текущее = ПОСЛЕДНИЙ замер (не макс/сумма):**
  - A2 reach: запиши **12** → текущее 12, % = (12−3)/(15−3) = **75%**. Затем запиши **8** (ниже) → текущее **8**, % = **42%** (прогресс может откатываться — это правильно).
  - A3 reduce: запиши **80** → % = (90−80)/(90−75) = **67%**. Запиши **75** (= цель) → **авто-завершение** (бейдж «Достигнута», 100%).
- [ ] **B3 maintain — «в коридоре N%» = доля замеров между границами:**
  - A4: было 50% (2/4). Запиши **8** (в коридоре) → 3/5 = **60%**. Запиши **6** (ниже) → 3/6 = **50%**. На графике точки относительно зоны коридора.
- [ ] **B4 правка записи (✎):** на A1 правка «+4» → «+10» → сумма 3+2+10=15 → **15/30 = 50%**; бар/график/вехи пересчёт **без F5**. Edge: правка вверх до цели → авто-завершение; правка вниз у завершённой ниже цели → авто-возврат в работу.
- [ ] **B5 удаление записи (✕, с подтверждением):** запись ушла → процент/график пересчёт без F5. Edge авто-возврата: доведи A3 до 75 (завершена) → удали запись 75 → текущее → 80 → <100% → цель **сама в работу**, форма записи вернулась.
- [x] **B6 график (ApexCharts) при ≥2 записях:** линия + маркеры + тултипы (дата+значение) + зум. accumulate — нарастающая; reach/reduce — замеры; maintain — замеры + **закрашенная зона коридора**. При 0–1 записи блок «Динамика» скрыт; у rollup-цели — тоже (прогресс из детей). _(·28: проверено — reach «Динамика, раз» 5→8→10 с зумом; maintain «Динамика, ч» с закрашенным коридором 7–9.)_
- [ ] **B7 пустая/нечисловая запись:** пусто → не отправляет / понятная ошибка (не молчит); буквы/отрицательное → ошибка «значение числом», мусор не записывается, не падает.

## D. Подцели + rollup + навигация

- [ ] **D2 «+ Подцель» на цели С записями** → сразу модалка «Подцели недоступны» (форма НЕ открывается).
- [ ] **D3 rollup** — у родителя % = среднее % подцелей; «N из M подцелей»; своя форма записи скрыта.
- [ ] **D4 переход в подцель** (клик) → экран меняется на подцель (без F5).
- [ ] **D5 «↑ К цели «…»»** — с подцели возврат к родителю (с именем родителя).
- [ ] **D6 глубина** — подцель подцели свыше лимита → ошибка 422 (понятная).

## F. Lifecycle

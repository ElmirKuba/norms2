# ADR-0023: Деплой и юрисдикция (J1, J2, B3, B6)

- **Статус:** accepted
- **Дата:** 2026-05-30
- **Решает:** Elmir (юрисдикция — его решение, [CLAUDE.md](../../CLAUDE.md) п.5)
- **Контекст-теги:** [deployment] [152fz] [ops]

## Решение

- **J1 — юрисдикция/хостинг:** на время разработки — хостинг **в РФ допустим**, домен `нормисы.рф`. Переезд вне РФ — **только под давлением** (блокировки/требования). Готовность к переезду сохраняется (портативность Docker).
  - **Почему приемлемо:** главный щит — отсутствие ПДн ([ADR-0001](./0001-data-minimization-no-pii.md)), а не локация; «вне РФ» нужно было для устойчивости, этот риск осознанно отложен.
  - **Корректирует** формулировки «хостинг вне РФ» в `Technical-assignment.md`, `CLAUDE.md`, приватном `152fz.md` → «РФ на время разработки, готовы к переезду».
- **J2 — reverse-proxy/TLS:** **Traefik** (docker-native, авто Let's Encrypt, минимум конфига в compose).
- **B6 — backup-домен:** **позже** (зафиксировать необходимость, купить перед/при давлении).
- **B3 — IP-логи:** дневная соль для `ip_hash` — в секретах/конфиге, ротация ежедневно. Sweep `security_logs` (TTL 60д) и истёкших `invite_codes` — через **`@nestjs/schedule`** (не pg_cron, чтобы не плодить инфру).

## Альтернативы
- Жёстко «только вне РФ с фазы 1» — дороже и медленнее на старте; отложено до давления.
- nginx (J2) — TLS вручную; Traefik проще для Docker. pg_cron (B3) — лишняя зависимость.

## Последствия
- `deployment.md` — Docker, Traefik+LE, ENV, бэкапы, sweep.
- Обновить «вне РФ» в TA/CLAUDE/152fz на новую формулировку.
- Перед боевым неотложенным релизом «под давлением» — выполнить переезд + backup-домен (план в `deployment.md`).

## Дополнение (2026-06-10, F6.2 — реакция на давление, решает Elmir)

Правовой проход F6.2 (сверка `/terms` и `/privacy` с кодом/ADR) уточняет реакцию на давление:

- **Снос и переезд — равноприоритетны, выбор по обстоятельствам.** Под внешним давлением/угрозой существованию проект может быть **либо перенесён** (вне РФ, портативность Docker — как в J1), **либо полностью прекращён с удалением баз данных**. Ни один вариант не «дефолт» — решение ситуативное. Это расширяет исходный J1 («переезд под давлением»): прекращение+снос — равнозначная альтернатива переезду.
- **Все участники предупреждены заранее** — в `/terms` (раздел 7 + §6 «Законные запросы») и `/privacy` (разделы «Локализация и хранение», «Законные запросы»): устойчивость и сохранность проекта не гарантируются.
- **Локализация в текстах переформулирована вокруг «нет ПДн»** (ADR-0001): требование локализации БД ПДн граждан РФ к проекту, как правило, неприменимо (ПДн не накапливаются); РФ-инфраструктура — лишь констатация текущего dev-хостинга, не твёрдое обязательство. Это снимает риск «ложной клаузулы» при переезде.
- **Раскрытие по законным запросам** оставлено, но с честной рамкой: раскрывать по сути нечего (логин/псевдоним/технические записи), профилирования нет.
- **Удаление аккаунта (Q1):** формулировка `/privacy` («данные удаляются либо обезличиваются») оставлена осознанно — по факту soft-delete (ADR-0017, `deletedAt`, узел дерева сохраняется), но регламента восстановления для пользователя нет → для него необратимо; Elmir счёл формулировку приемлемой.

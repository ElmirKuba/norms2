# ADR-0043: Проверка живости сессии в Guard — немедленный отзыв устройства

- Статус: принято (2026-06-09)
- Контекст: фаза 1, F3.6+. Отзыв сессии (кик устройства) не действовал сразу:
  `AuthGuard` доверял подписи access-JWT и проверял аккаунт + бан (ADR-0038), но
  **не проверял, жива ли сессия** (`sid` из токена, ADR-0041). Кикнутое устройство
  работало до истечения access-токена (TTL ~15 мин): продолжало банить, ходить в
  профиль/приглашения. Refresh-путь уже безопасен (`rotateSession` отвергает
  `revokedAt`/истёкшие), дыра — только в Guard.

## Развилка (ToT) — как сделать отзыв мгновенным

| Ветвь | Суть | Минус |
|---|---|---|
| **A. Проверка сессии в Guard** | `sid` → живая сессия на каждый запрос | access перестаёт быть «чисто stateless» (но бан-чек уже stateful) |
| B. Короткий TTL access (15с–1мин) | окно отзыва = TTL | refresh-шторм, всё равно есть окно |
| C. Денилист в Redis | O(1) отзыв | +зависимость Redis, преждевременно для self-hosted |
| D. WebSocket/SSE-пуш | мгновенный UX | **не enforcement** (клиент может проигнорить); +инфра WS |

## Решение

**Ветвь A** (реш. Elmir 2026-06-09). Guard проверяет `SessionDomainService.isActive(sid)`
(сессия есть, не отозвана `revokedAt IS NULL`, не истекла) на каждый запрос → отзыв
действует **немедленно** (кикнутый падает на следующем же действии: 401 «Сессия
завершена.»). TTL не трогаем; новой инфры нет — Guard и так ходит в БД (аккаунт +
бан), это +1 индексный lookup.

Refresh уже отвергает отозванные/истёкшие сессии, поэтому кикнутый не перевыпустит
токен по cookie. Фронт изменений не требует: 401 от Guard → интерсептор пробует
refresh → refresh падает (сессии нет) → `clear` + редирект на `/login`.

**D (WebSocket-пуш)** — опционально позже как UX-слой поверх A (мгновенный редирект
без действия), но не замена серверной проверке.

## Граница модулей — `SessionCoreModule`

`SessionDomainService` нужен теперь и Guard'у (в `AccessControlModule`), и контроллеру
`/sessions` (`SessionsModule` импортирует `AccessControlModule`). Прямая зависимость
замкнула бы граф. Поэтому ядро вынесено в **`SessionCoreModule`** (репозиторий +
`SessionDomainService`, без зависимости от AccessControl) — зеркало `BanCoreModule`
(ADR-0038): `AccessControlModule` и `SessionsModule` импортируют ядро; цикла нет.

## Последствия

- Отзыв устройства/«выйти на остальных»/деактивация/удаление аккаунта (revoke
  сессий) теперь рубят доступ немедленно, а не в окне TTL.
- Access-токен фактически stateful (как и при бан-чеке) — осознанный размен ради
  безопасности; стоимость — один индексный запрос на запрос.
- `+ existsActiveById` (repo/port) и `isActive` (domain).

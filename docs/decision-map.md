# decision-map.md — карта решений по проекту «Нормисы»

> **Назначение.** Полный реестр точек, где требуется принять решение — по концепту, философии, архитектуре, БД, API, фронту, деплою. Проходим **по шагам**, каждое решённое — переезжает в [`decisions/`](./decisions/README.md) как ADR.
>
> **Зачем отдельный файл, а не `todo.md`.** `todo.md` = план *работ* (что кодить). Этот файл = карта *решений* (что выбрать до кода). Он resumable: если кончатся лимиты — открываем его, смотрим «Текущую позицию» и продолжаем. Указатели на него есть в [`../CLAUDE.md`](../CLAUDE.md) (грузится всегда) и в [`../todo.md`](../todo.md).
>
> **Метод.** Каждый пункт — это Tree-of-Thoughts ([`methodology.md`](./methodology.md)): 2–4 ветви → оценка (слои / обратимость / 152-ФЗ / простота / цена сопровождения) → выбор → ADR.

## Легенда статусов

- ⬜ — не обсуждали
- 🔄 — обсуждаем сейчас
- ✅ — решено (ссылка на ADR)
- ⏸️ — сознательно отложено (с указанием, до какой фазы)

## Порядок обхода (рекомендуемый)

Сначала то, что разблокирует написание инженерных доков и кода:

1. **D** (доменная модель) — D2→D3→D4, затем D1,D5–D8. Разблокирует `domain-model.md`.
2. **E** (БД) — следствия из D. Разблокирует `database.md`.
3. **C** (архитектура) — C1 (токены) критичен. Разблокирует `architecture.md`.
4. **F** (API) — следствия C+D. Разблокирует `api-contracts.md`.
5. **G/H/I** (backend/frontend/ui) — правила и экраны.
6. **J/K** (деплой/локалка) — параллельно с концом Фазы 1.
7. **A** (концепт/философия) — можно обсуждать в любой момент, влияет на копирайт и ToS, но не блокирует код ЛК.
8. **B** — операционные доводки приватности по мере написания БД/деплоя.

Каждый закрытый пункт: статус → ✅ + ссылка на ADR, и обновить «Текущую позицию» вверху.

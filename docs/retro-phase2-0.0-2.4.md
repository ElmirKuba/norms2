# Ретро 0.0 → 2.4 (три шляпы) — 2.4·22

> Обзорный проход по готовому функционалу (фаза 1 ЛК + фаза 2.0→2.4 «Акцент»: привычки,
> адаптивная лесенка, задачи дня, стартовый пак, микро-победы) глазами **бизнес-аналитика +
> системного аналитика + эксперта по помощи людям**. Не блокирует разработку — это меню
> находок: из него выбираем форки/задачи. Пункты, проверенные по коду, помечены ✔(verified).

## P1 — стоит закрыть раньше прочего

1. **Локаут без восстановления** (помощь людям + БА) — ✔verified.
   Восстановление по секретным вопросам **необязательно и не предлагается** на регистрации
   (`register.dto`: login/alias/password/inviteCode, без recovery; вопросы задаются позже в
   настройках, по желанию). Итог — пользователь блокирует себя сам: **вчера так потерян
   `@horstman56`** (забыл пароль, вопросы не задал → пришлось удалять аккаунт).
   _Варианты:_ (а) мягкий шаг онбординга «задай 2 вопроса восстановления» после регистрации;
   (б) ненавязчивый баннер-напоминание, пока не задано; (в) сделать частью регистрации.
   Быстро-средне, **высокая ценность** (прямой отток).

## P2 — важное, но не горит

2. **Гонка на записи 2.4 при мультидевайсе** (СА) — ✔verified.
   На `habits`/`tasks` **нет `version`-колонки**, оптимистичный лок (ADR-0035) для accent **не
   подключён** (в `modules/accent`/`repositories/accent` — ни optimistic/retry/CAS). Лесенка
   двигается read-modify-write (`easyStreak`/`missStreak`) на каждом complete. Два устройства
   разом → потерянное обновление / неверный шаг планки. _Решить:_ атомарный апдейт счётчиков
   (CAS) или version+retry для ladder/complete.

3. **Вкладка «Цели» — заглушка** (БА) — ✔verified.
   В нав-меню «Акцента» есть «Цели», но **goals-модуля на бэке нет** (роут во фронте есть).
   Пользователь видит мёртвую вкладку. _Решить:_ либо строить (плановая подфаза), либо пока
   **скрыть/пометить «скоро»**, чтобы не показывать пустышку.

4. **Серии / «never miss twice» не реализованы** (помощь людям).
   Ядро anti-burnout-обещания («не обнуляться») пока частичное: есть `donePercent` и адаптивная
   лесенка, но самой **серии нет** (на «Сегодня» висит плейсхолдер «🔥 серия — скоро»;
   recovery-ветка лесенки и overall-streak отложены ~2.8). Это ключевой смысловой пробел.

5. **RPG-атрибуты собираются, но мертвы** (помощь людям/БА).
   В форме привычки выбираешь «прокачивает атрибуты», но очки/прогрессия отложены (~2.9) —
   выбор есть, отдачи нет (ожидание ≠ результат). _Решить:_ пометить «скоро» или приглушить,
   пока нет геймификации.

6. **Приватность инвайтов** (СА / 152-ФЗ).
   Текст-причина приглашения виден приглашённому; логины/псевдонимы видны в пределах своего
   «поддерева». При позиции «без ПДн» — решить, приемлемо ли (флажилось ранее как смелл).

## P3 / зафиксировано

7. **Полночный rollover (cron)** отложен (~2.9) — задачи материализуются лениво на чтение;
   серии без rollover не поедут.
8. **Тестов нет (jest не пишем)** — верификация live-прогонами (решение Elmir). Тех-долг, ок пока.
9. **Прод-хардненинг** — root-пароль в открытую (дев-стенд); для реального прода ротация + SSH-ключи.
10. **Клавиатура vs нудж на мобиле** — нудж по IntersectionObserver не вычитает высоту клавиатуры
    (`visualViewport`); мелочь, отложено.

- **Крупное:** #2 (конкурентность), #4 (серии), #3-полное (построить «Цели»).

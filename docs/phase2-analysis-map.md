## 0. Видение Фазы 2 (захвачено нашими словами — НЕ потеряется, даже если все источники удалить)

«Акцент» — **личная ОС достижений** в духе SuperBetter + Tiny Habits (BJ Fogg). Ядро:

1. **Сверх-низкий порог входа.** 1 отжимание = засчитанная тренировка. Цель не «100 отжиманий», а «сделай 1, остальное бонус». Импульс важнее объёма.
2. **Лесенка прогресса.** Сегодня 1, завтра 2, через 11 дней 10+1 — система ведёт по микро-шагам к крупной цели, подбирая следующий чуть-больший шаг.
3. **Всё засчитывается.** Даже стакан воды → победа/очко. Любое микро-действие можно зафиксировать и получить признание.
4. **Геймификация:** очки, серии (streaks), достижения, уровни — как поддержка мотивации, не казино.
5. **Целеполагание под контролем:** цели → задачи → привычки/анти-привычки, с серверной агрегацией прогресса и идемпотентностью отметок.
6. **Круг доверия = инвайт-дерево Нормисов.** «Три близких, кому доверяешь» — маленький круг подотчётности. В фазе 2 — соц-фичи осторожно/опционально.

**North star:** даже демотивированный человек, делая смешно-маленькие шаги, забирает контроль над жизнью. Бесплатно, opensource, «для своих».

> Приоритетный артефакт против потери контекста. Если сессия оборвётся — Фаза 2 стартует отсюда.

## 5. Леджер идей (растёт по мере чтения — «память» между сессиями)

### Механики геймификации
_(из `gamification.md` — идеи, переформулированы; конкретные числа берём как стартовые наброски, перетюним под наше видение «всё засчитывается»)_
- **Геймификация поверх, не вместо.** Очки — отражение прогресса, НЕ валюта: нельзя потратить, нет магазина, нет pay-to-win, нет gate привилегий. Ачивки за осмысленное, не за гринд. (совпадает с нашим «не казино»)
- **XP за действия** (старт-наброски): задача 10; задача из цели +5; привычка/день 8; день без анти-привычки 8; дневник дня 5; серия 7д +20; серия 30д +100; цель 100% → 200 разово.
- **Уровни** = от суммарного XP, формула `level = floor(sqrt(totalXP/100))+1` (0/100/400/900/1600…). Чисто визуальный статус, без привилегий.
- **Серии (streaks):** последовательные дни; пропуск → сброс в 0; отображение 🔥N на карточке. **Freeze:** 1 заморозка/неделю спасает серию, не накапливается (use-it-or-lose-it). ⟶ хорошо ложится на «не наказывать за срыв».
- **Достижения** (примеры идей): Первый шаг (1-я задача), Достигатор (1-я цель), Неделя силы (7д), Железная воля (30д), Мастер привычек (5 привычек × 14д), Победитель (анти-привычка 30д), **Возвращение (вернуться после обрыва 7+д)** ← важно для «опущенного Васи», Сова/Жаворонок (время суток).
- **Анти-читинг (ВАЖНО, ложится на наши принципы):** XP только сервером; идемпотентность отметки (повтор не начисляет); дневной лимит источников (дневник 1/день); бэкдейтинг фиксирует факт, но XP не даёт. ⟶ согласуется с фазой-1: серверная агрегация + идемпотентность.
- **Модуль `gamification` подписан на доменные события** других модулей и начисляет награды; не дублирует прогресс, а читает события. ⟶ архитектурная идея: event-driven, отдельный модуль в нашей слоистой структуре.

> ⚠️ Файл `gamification.md` оказался усечён (строки 70–79 — повтор «Связь с другими модулями» + плейсхолдеры «усечено»). Раздел про связь с модулями в источнике неполный — учтём при синтезе (свяжем с goals/tasks/habits сами).

### Цели / задачи / привычки / анти-привычки
_(из `Technical-assignment.md` акцента — идеи переформулированы; их «фазы 0-9» и стек игнорируем, у нас свой фундамент)_
- **Состав сущностей-кандидатов:** `Goal` (большая измеримая цель), `WeeklyGoal` (недельный фокус), `Task` + `TaskTemplate` (повтор по RRULE), `AntiHabit` (таймер «держусь»), `Workout` (упражнения→подходы→повторы), `PointEvent` (идемпотентный факт очков), `Achievement`, `DayLog` (снимок дня: задачи/%/дневник).
- **Goal:** title, категория (напр. spiritual/physical/mental — у нас категории сделаем настраиваемыми, не навязанными), unit (км/раз/страницы), targetValue/currentValue, deadline. Прогресс = current/target. Записи прогресса = `GoalEntry` (дата, значение, заметка).
- **Task↔Goal:** выполнение задачи, привязанной к цели, даёт прогресс цели. Приоритет low/normal/high влияет на очки.
- **TaskTemplate = RRULE** (пн/ср/пт и т.п.); конкретные `Task` материализуются планировщиком на дату. День без материализации по RRULE серию НЕ сбрасывает.
- **AntiHabit = timer-модель** (не счётчик отметок): держусь N дней непрерывно; «Рецидив» сбрасывает таймер и растит attemptNumber; рекорд обновляется при рецидиве. Milestone-очки на 3/7/14/30… через отложенные джобы (у нас — `@nestjs/schedule`/очередь, без BullMQ если можно проще).
- **WeeklyGoal:** промежуточный горизонт между днём и большой целью, сбрасывается в понедельник.
- **DayLog:** снимок дня — основа серий и недельных сводок; % выполнения дня.
- **Workout:** структура упражнение→подход(вес,повторы); история для прогрессии весов; завершённая = очки. ⟶ кандидат отложить на под-фазу (сложнее остального).

### Прогресс-лесенка и микро-шаги
- **Конкретика вместо размытости:** каждая цель измерима, каждая задача выполнима «за один тап» (≈30 сек отметить). Это и есть техническая опора нашего «1 отжимание = победа».
- **Восстановление после срыва (comeback) — продуктовый принцип №1 для нашего «Васи»:** пропуск помогает вернуться, а не штрафует деморализующе. Метрика успеха = «после срыва возвращается, а не бросает».
- **Чем НЕ todo-list:** todo показывает невыполненное (чувство вины), «Акцент» показывает прогресс/рост (эмоция). Серии создают инерцию «не хочу терять streak».
- _(механику авто-подбора следующего чуть-большего шага — 1→2→…→100 — источник явно НЕ описывает; это НАШЕ усиление, спроектируем сами в синтезе → развилка ниже)_

### Открытые развилки Фазы 2 (→ ADR в батче G)
- **R1 Категории целей:** навязанные (spiritual/physical/mental) vs свободные пользовательские? (склоняемся к свободным + пресеты).
- **R2 Авто-лесенка прогрессии:** как система подбирает следующий шаг (1→2→10+1)? Линейно/адаптивно/вручную? — ядро нашего видения, источник не покрывает.
- **R3 Workout-модуль:** в scope фазы 2 или отложить как под-фазу 2.x (он самый тяжёлый).
- **R4 Realtime:** источник хочет WebSocket для мгновенных очков. Нужен ли нам в фазе 2 или хватит оптимистичного UI + REST? (склоняемся ко второму на старте).
- **R5 Соц/круг доверия:** источник = строго одиночный (юзеры не видят друг друга). Наше видение хочет «круг доверия» через инвайт-дерево. Развилка: оставить одиночным в фазе 2, соц — отдельной под-фазой?
- **R6 timezone/locale на аккаунте:** у Акцента всё завязано на TZ юзера (ролловер дня, серии по локальной полуночи) и locale (RU/EN). В нашем `accounts` (фаза 1) этих полей НЕТ. Развилка: добавить `timezone`/`locale` в accounts, или хранить в настройках раздела? (locale у нас = RU по [ADR-0022](./decisions/0022-concept-and-philosophy.md), но TZ для ролловера нужен реально).
- **R7 Состав MVP фазы 2:** источник тянет 10 под-модулей (goals/tasks/anti-habits/weekly/metrics/lessons/workouts/scheduled/gamification/dashboard). Для нашего MVP «Акцента» — что в первую очередь, что отложить? (кандидат-ядро: goals + tasks + gamification + лесенка; остальное волнами).

### Доменные сущности — детальные поля (из `domain-model.md`, переформулировано; uuid/email/bcrypt → у НАС: id `uuidv7___unixmillis`, аккаунт уже есть в ЛК фазы 1, не дублируем User)
_Важно: их `User` = наш `accounts` из фазы 1. НЕ заводим заново — все сущности Акцента вешаем на `account_id`. timezone/locale: в фазе 1 их у accounts нет → развилка R6._
- **Goal:** account_id, title, category, unit (свободная строка: «км/раз/страниц/дней»), targetValue, currentValue (кэш = Σ entries), deadline?, status (active/completed/archived), completedAt?. Инвариант: current≥target → completed (не удаляется); target>0.
- **GoalEntry:** goalId, value (можно отрицательное — коррекция, ≠0), note?, occurredOn (дата в TZ юзера), createdAt.
- **TaskTemplate:** account_id, title, description?, icon?, priority (low/normal/high → влияет на очки), recurrence (RRULE-объект: freq/byweekday/interval), active.
- **Task:** templateId? (null=разовая), account_id, title, date, status (pending/done/skipped), completedAt?, skipReason? (postponed/cancelled). Уник (templateId,date) — 1 инстанс шаблона на день.
- **AntiHabit (timer):** account_id, title, description?, startedAt, attemptNumber, longestStreakDays. Серия=floor((now−startedAt)/сутки); рецидив→обновить рекорд, startedAt=now, attempt++.
- **AntiHabitDay** (опц., история): antiHabitId, date, state (held/relapsed).
- **WeeklyGoal** + **WeeklyGoalItem:** недельный фокус (weekStart=пн) + чек-айтемы (text, done).
- **DailyMetric:** account_id, date (уник с account), sleepHours? (0..24, шаг .5), energy? (1..10), mood? (1..10), note?.
- **DailyLesson:** account_id, date, content (markdown), tags[].
- **Workout/Exercise/WorkoutSession/SetLog:** шаблон трен.→упражнения(targetSets/Reps,lastWeight,order); сессия→подходы(setNumber,reps,weight,rpe?). [тяжёлый модуль — R3].
- **ScheduledAction:** account_id, title, dueDate, remindDaysBefore?, status(open/done) — разовое дело с напоминанием.
- **PointEvent:** account_id, source(enum), sourceId(строка), points, occurredOn, revokedAt?. Уник (source,sourceId) среди не-revoked = идемпотентность.
- **Achievement** (каталог: code PK, title/desc/icon) + **UserAchievement** (account_id, code, awardedAt; уник (account,code)).

### UX-сценарии (из `ui-ux.md`, переформулировано; их палитра/Tailwind НЕ берём — у нас своё ([ADR-0025](./decisions/0025-ui-ux-design-language.md): тёмная+toggle, SCSS, MatDialog))

- **a11y:** prefers-reduced-motion отключает анимации; ARIA на кастомных; контраст ≥4.5 — взять как требование.

Срыв → «Comeback начинается сегодня» (не «серия прервана»); задача не сделана → «серия ждёт продолжения»; пустой экран → «день начинается — первый шаг?»; срыв анти-привычки → «новая попытка». Достижения **частые в начале** (первая задача/неделя/уровень), редкие позже — имитируют кривую нарастающего интереса до того, как мотивация включилась.
- **«Минимум плохого дня» (= наш `minVersion`/лесенка вживую):** вечером, если задача не сделана и серия под угрозой — карточка показывает мини-версию («Минимум сегодня: 5 мин растяжки»). Данные уже в задаче, без доп.запроса. ⟶ прямой кирпич для R2.
- **«Урок дня как якорь»:** фраза, которую юзер написал себе сам из ресурсного состояния, показывается когда он не в ресурсе (открыл вечером вместо YouTube). Не цитата из интернета — твои слова тебе. Сильная антирецидивная механика.
- **Anti-habit экран:** большой живой таймер дн:ч:мин:сек (фронт `setInterval`), circular ring = currentDays/targetDays, «Рецидив» с подтверждением+заметкой, история попыток.

### Новые развилки из добора
- **R8 Серии:** считать on-demand (window function) или материализовать (`streak_state` + обновление)? Старт — on-demand, материализация при росте данных.
- **R9 Напоминания:** in-app vs web-push (email у нас нет). Старт — in-app; push — отдельной волной.
- **R10 Атрибуты vs категории целей** (переопределяет R1): брать RPG-атрибуты (6 шт + паучья диаграмма) вместо/вместе с категориями? Склоняюсь: настраиваемые атрибуты + визуализация баланса — сильнее и интереснее, ложится на «прокачку персонажа».
- **R11 Формула XP/уровня:** sqrt (акцент) vs `100*N^1.5` (спек) vs своя. + множитель серии (×2 при 100д). Решить в синтезе с учётом «не казино».
- **R12 Частичное выполнение + типы привычек** (бинарная/количественная/временная): включать в MVP? Прямо связано с R2-лесенкой (частичное = микро-победа).
- **R13 Вехи целей (milestones):** в MVP или волной? (мощно для длинных целей, но усложняет).
- **R14 Онбординг:** делать ли управляемый онбординг (выбор сфер→первая цель→первое достижение) в фазе 2 — критичен для «опущенного Васи» (первая победа за 5 мин).
- **R15 Доп-модули (Прил. D):** какие из шаблоны/квесты/сезоны/благодарность/колесо-баланса/ритуалы/pomodoro/mind-map/карточки берём в Акцент и когда. Большинство — поздние волны; кандидаты в раннее: шаблоны (снижают порог) + мотив-карточки (= урок-якорь).
- **R16 Экспорт данных** (JSON/CSV) — «данные пользователя» — заложить как принцип (согласуется с нашим opensource/приватность), реализация — волной.
- **R17 State-aware ядро (КРУПНОЕ решение):** брать ли модель состояний (survival/recovery/growth/…) + check-in + автоснижение нагрузки? Это главное отличие SuperBetter от обычных трекеров и прямо служит «опущенному Васе». Развилка: включить лёгкую версию в MVP (check-in настроения/энергии → подстройка рекомендаций) или волной. Склоняюсь: лёгкий check-in + recovery-режим важны рано.
- **R18 Каркас сущностей:** наши «цели/задачи/привычки» (из акцента) vs SuperBetter «EpicWin/Quest/PowerUp/BadGuy/Ally/Identity». Слить? Маппинг: EpicWin≈Goal(+why,+fallback,+вехи), Quest≈Task, PowerUp=новый тип «микро-победа» (ядро лесенки!), BadGuy=анти-привычка-как-враг, Ally=круг доверия, Identity=профиль/псевдоним (у нас уже alias!). Это центральное решение синтеза.
- **R19 Bad Guys (препятствия как сущность):** брать ли модель «врагов с контрмерами» вместо/вместе с простыми анти-привычками? Мощно мотивационно, но усложняет. 
- **R20 Secret Identity:** у нас уже есть `alias`-псевдоним из ЛК — можно развить в игровую личность (класс/архетип/миссия) для Акцента. Дёшево и сильно (анти-стыд).
- **R21 Accessibility-first:** заложить screen-reader/keyboard/voice/low-cognitive-load в ui-ux Акцента с самого начала (а не «фаза 9»). Для нашей аудитории (Вася в плохом состоянии) — критично.
- **R22 AI-слой:** Нормисы про «ИИ помогает миру» — AI-декомпозиция целей в микрошаги/перепаковка срывов реально усиливает продукт. Когда? (поздняя волна, но заложить границы безопасности из спека сразу).

### Социальное / круг доверия (связь с инвайт-деревом)
- **Спек (20):** соц — мощный мотиватор, но источник токсичности → опционально и осторожно; кор работает в одиночку. Функции-кандидаты: друзья видят прогресс **с согласия**; лиги (анонимные группы); совместные челленджи; менторство (опытные→новичкам).
- **Наша адаптация (R5):** «круг доверия» = инвайт-дерево Нормисов. Видеть прогресс приглашённых/пригласителя (с согласия), поддержка, совместные челленджи в малом доверенном круге. Менторство ≈ пригласитель помогает приглашённому. Без публичных лиг/токсичной конкуренции. Соц — отдельной волной после одиночного MVP.

### Идеи из SuperBetter/LifeOS-спека (прочитан ВЕСЬ, 932 стр, покрытие 1-320/308-647/640-931 без пропусков; переформулировано; НЕ удалён — бэкап Elmir)
**Каркас SuperBetter (6 канонических сущностей — мощная альтернатива/надстройка над целями-задачами):**
- **Epic Win** — вдохновляющая большая цель (позитивная формулировка, success_criteria, fallback-версия на плохой день, вехи, ≥1 микро-успех 1-5 мин). ≈ наш Goal, но с «why_it_matters» и fallback.
- **Power-Up** — смешно-маленькое действие 10сек-5мин, не требует воли, выполнимо в плохой день, фиксируется как победа (типы: физ/эмоц/мент/соц/сенсор/быт). = техническая форма «1 отжимание = победа».
- **Quest** — действие к цели (5-30 мин, разовое/повторяемое, difficulty 1-5). ≈ Task.
- **Bad Guy** — препятствие как сущность (внутр.критик, прокрастинация, тревога, думскролл, недосып…): name/trigger/symptoms/intensity/counterplay. 🆕 мощно — анти-привычки как «врагов с контрмерами».
- **Ally** — союзник поддержки (witness/supporter/accountability/emergency/coach) с consent_scope. → ложится на наш «круг доверия»/инвайт-дерево.
- **Secret Identity** — игровая личность (имя героя/класс/архетип/миссия) для психодистанции «я не ленивый — я игрок в тяжёлом квесте». 🆕 сильная анти-стыд механика.

**State-aware ядро (ГЛАВНОЕ отличие от акцента — адаптация под состояние):**
- **Состояния пользователя:** survival/recovery/stability/growth/sprint/maintenance (+ перегружен/откатился). Check-in (mood/energy/pain/sleep/anxiety/focus) → система выводит состояние → меняет рекомендации.
- **Автоснижение нагрузки** при ухудшении: меньше плана, снять квесты, упростить вход, убрать стыд, переформулировать цель на 24ч. → «не наказывать Васю в плохой день» технически.
- **Режим паузы** всей системы (отпуск/болезнь) — серии замораживаются.
- **Алгоритм рекомендаций** (псевдокод 12.2): по состоянию выдаёт 1-3 микро-действия (survival→2 power-ups+1 квест≤5мин; growth→больше квестов).

**Домены жизни** (health/sleep/sport/work/money/relationships/learning/home/creativity/purpose) — цели группируются по доменам (≈ атрибуты/категории, см. R10).

**Сценарии тяжёлых состояний** (депрессия/боль/инвалидность/тревога/ADHD/выгорание/зависимости) — для каждого свой паттерн микродействий. → ценные шаблоны контента (волной).

**Принципы (берём как наши):** «сначала безопасность→потом мотивация; микро-действие→амбиция; восстановление→масштаб; доверие→данные». Anti-метрики: НЕ оптимизировать время-в-app/пуши/залипание (вредно для recovery). Путь «недовольного Васи» (20): вернуть дееспособность→микро-победы→ритм→рост→профессионал→лидер/владелец. = ровно твоё видение.

**Accessibility-first:** screen-reader, high-contrast, keyboard, large tap, voice-input, low cognitive load, простой язык, offline-core, dark/light. → заложить как требование Акцента.

**AI-слой (поздняя волна, с границами):** декомпозиция целей в микрошаги, перепаковка срывов в recovery, next-best-action — НО нельзя: усиливать зависимость, стыдить, обещать лечение, подменять врача, скрытые манипуляции. → согласуется с нашим «ИИ помогает, не вредит».

**Этика/риски:** токсичная продуктивность, стыд, игрофикация боли, app-зависимость, подмена терапии. Защита: режим без наказаний, выключаемая соревновательность, кризисные контакты, экспорт+удаление данных.

**Roadmap-идея (5 фаз):** MVP(checkin/domains/powerups/quests/epicwin/streaks-без-наказания/allies)→recovery-aware(state machine/bad guys)→social/squads→AI→scale. → ориентир для волн нашего Акцента.

- **Антихрупкость:** срыв — часть пути, не наказание, помощь восстановиться + извлечь урок (1.2 П5, 7.3).

- **Вехи целей (Milestones)** (5.3): большая цель → промежуточные вехи, каждая = мини-достижение с наградой. (в акценте целей-вех не было) → новая под-сущность Goal.
- **Анализ триггеров срывов** (7.4): фиксировать триггер срыва, показывать паттерны (когда/в каких ситуациях чаще). Усиление анти-привычек.
- **Психо-механики (12):** дофаминовая петля (действие→мгновенная награда), эффект Зейгарник (80%-бары тянут к 100%), якорение (after X→Y, цепочки привычек), принцип Премака (приятное как награда за нужное), прайминг окружения.
- **Формулы (16):** уровень `XP(N)=100*N^1.5`; множитель серии `1+min(streak/100,1)` (макс ×2 при 100д). — альтернатива sqrt-формуле из акцента, выбрать в синтезе.
- **Онбординг (19):** выбор 1-3 сфер → первая цель (с подсказками) → первая привычка (из шаблонов) → достижение «Начало пути» сразу → тур. Прогрессивное раскрытие сложных фич.
- **Режим паузы (26.2)** для отпуска/болезни — серии замораживаются, не наказывает (шире, чем пауза цели из акцента — пауза всей системы).
- **Доп. модули-идеи (Прил. D):** шаблоны целей/привычек; квесты (цепочки задач с общей наградой); сезоны/челленджи (30-дневки); дневник благодарности; колесо баланса жизни; ритуалы (утро/вечер — наборы привычек); Pomodoro-фокус; mind-map целей; мотивационные карточки (= наш «урок-якорь»).
- **Экспорт данных** (JSON/CSV) — «данные принадлежат пользователю» (18.3, 28).

**Anti-цели продукта (Прил. F) — берём как наши принципы:** не вызывать зависимость ради зависимости; не красть время бесконечным контентом; не стыдить за неудачи; не быть источником тревоги; не продавать данные; не навязывать чужие цели. + «экономия внимания» (22): минимум времени в приложении — максимум действий в жизни. Главная метрика — реальные изменения в жизни, не DAU (27.3).

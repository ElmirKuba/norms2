# ADR-0058: Полярность лесенки привычек + `clock`-kind (время-цель)

- Статус: принято (2026-07-06, фаза 2, подфаза 2.6 · Трек B / FEAT-H2, реш. Elmir) — **обратимо**
  (аддитивно: дефолт `direction='raise'` = текущее поведение; `clock` — новое опц. значение kind).
- Контекст: `HabitLadder` (`minTarget`/`currentTarget`/`goalTarget`/`step`/`policy`) и `LadderEngine`
  (`nest/src/modules/accent/habits/domain-services/accent-ladder-engine.domain-service.ts`) сейчас
  жёстко «**выше — лучше**»: успех = `performed ≥ currentTarget`, серия лёгких → планка растёт ВВЕРХ
  к `goalTarget`; инвариант `minTarget ≤ currentTarget ≤ goalTarget`. Кейс «сдвиг окна сна» (отбой
  ≤ 3:00 → 2:30 → 2:00) — **зеркальный**: успех = «раньше/меньше», планка должна двигаться ВНИЗ
  (ужесточение дедлайна) — это «reduce»-семантика, которой у привычек нет. Плюс время после полуночи
  (03:00, 02:30) требует представления «минуты от вечернего якоря», а не сырого числа, и сравнения
  через полночь. У ЦЕЛЕЙ асимметрия уже решена (`GoalDirection` incl. `reduce`, [ADR-0052]) — у
  привычек нет.

## Решения
1. **Полярность лесенки: `HabitLadder.direction: 'raise' | 'lower'`** (дефолт `'raise'` = текущее).
   При `'lower'`: успех = `performed ≤ currentTarget`; «подъём» лесенки = `currentTarget − step`
   (к `goalTarget`); откат = `currentTarget + step` (к `minTarget`); инвариант **переворачивается**:
   `goalTarget ≤ currentTarget ≤ minTarget`. В `LadderEngine._apply` — зеркальная ветка (инверсия
   сравнения и знака шага); «минимальная победа держит серию» и RRULE-расписание переиспользуются.
2. **Новый `HabitKind = 'clock'`** — целевое ВРЕМЯ суток / дедлайн. Значения лесенки хранятся как
   **«минуты от вечернего якоря»** (`anchorMinutes`, напр. 18:00 → 0), корректно проходит полночь;
   ввод факта = фактическое время → в те же минуты. `anchorMinutes` — в тот же jsonb `ladder`.
   Обычно идёт с `direction='lower'` (двигаем отбой раньше), но оси независимы.
3. **Аддитивно, без слома:** `ladder` — jsonb (новые поля без миграции структуры); `kind` — varchar
   (новое значение). Миграция минимальна: дефолт `direction='raise'` + опц. `anchorMinutes`.
4. **Рассмотренные альтернативы (ToT):**
   - **A (отдельная сущность «время-привычка»)** — отвергнута: дублирует всю механику привычек
     (расписание, задачи, серии, минимальная победа).
   - **B (хранить сырое время суток без якоря)** — отвергнута: переход через полночь ломает сравнение
     и знак шага (02:30 «меньше» 23:00 численно, но позже по циклу дня).
   - **C (полярность на лесенке + `clock`-kind с якорем)** — **выбрано**: переиспользует привычки/
     задачи/RRULE/минимальную победу; аддитивно; концептуально симметрично целям.

## Последствия
- Привычки покрывают reduce/дедлайн-семантику (сон «не позже», меньше экранного времени и т.п.).
- `LadderEngine` получает зеркальную ветку — **риск не сломать `raise`**: обязательна регресс-проверка
  существующих привычек (шаг H2·6b).
- Фидбэк лесенки инвертируется в копирайте («сдвигаем отбой раньше: 2:30», не «провалил»).
- Реализация — Трек B подфазы 2.6, шаги **H2·2…H2·6** (`impl-phase2-plan.md`).
- Связано: [[0052-accent-goal-direction-and-computed-progress]] (та же асимметрия у целей),
  [[0035-concurrency-control]] (CAS лесенки), `gamification §7` (алгоритм `LadderEngine`).

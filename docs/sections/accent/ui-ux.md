# Акцент — UI/UX (Фаза 2)

> Экраны, дизайн-язык, доступность, тон. Наследует дизайн фазы 1 ([ADR-0025](../../decisions/0025-ui-ux-design-language.md)): тёмная тема + переключатель, **чистый SCSS** (без Tailwind), свои компоненты, модалки через **MatDialog** ([ADR-0026](../../decisions/0026-modal-system.md)); графики прогресса — **ApexCharts** за своим wrapper `app-chart` ([ADR-0055](../../decisions/0055-progress-charts-apexcharts.md)). Эндпоинты — [`api-contracts.md`](./api-contracts.md). Тон/принципы — [README §6](./README.md).

## 1. Тон: «интерес, не страх» (критично)
Каждый текст двигает вперёд, не давит назад. Это не косметика — это механика удержания «Васи».

| Ситуация | ❌ страх | ✅ интерес |
|---|---|---|
| Задача не сделана | «Не выполнено» | «Серия ждёт продолжения» |
| Серия сброшена | «Серия прервана 😔» | «Comeback начинается сегодня» |
| Цель отстаёт | «Отстаёшь от плана» | «12 / 77 — продолжаем» |
| Пустой экран | «Нет данных» | «День начинается — первый шаг?» |
| Срыв анти-привычки | «Срыв» | «Новая попытка» |
| Плохой день (recovery) | — | «Сегодня хватит минимума. Это уже победа» |

## 2. Дизайн-язык
- Тёмная по умолчанию + toggle (CSS-переменные, как фаза 1).
- Минимал с теплотой: мягкие скругления, тёплый акцент, воздух. Цифры читаются «с метра» (% дня, серия, уровень).
- **Один экран — одно главное решение. Один день — один главный прогресс.**
- Мгновенный отклик: оптимистичный UI, toast «+N», анимация чекбокса. Ошибка → откат + тост.

## 3. Доступность (accessibility-first, R21 — с самого начала, НЕ «потом»)
Аудитория включает людей в тяжёлом состоянии (депрессия/тревога/ADHD/боль), поэтому a11y — в MVP:
- клавиатурная навигация, видимый focus-ring; ARIA-роли на кастомных компонентах;
- контраст ≥ 4.5:1 (мелкий) / 3:1 (крупный);
- `prefers-reduced-motion` отключает нерезкие анимации;
- простой язык, низкая когнитивная нагрузка, крупные тап-таргеты;
- один главный шаг на экран (особенно важно для ADHD/перегрузки);
- кандидаты (волной): голосовой ввод, offline-core.

## 4. Карта экранов (lazy feature `accent`)
```
/accent                       дашборд (главный)
/accent/checkin               чек-ин состояния + тренды самочувствия 7/30/90 (из CheckIn)
/accent/goals  /goals/:id     цели (список / деталь: % + прогресс-бар, график динамики, прогноз, вехи)
/accent/habits                привычки + дневной чеклист (вкладки Сегодня / Шаблоны)
/accent/micro-wins            микро-победы (быстрый список «сделать сейчас»)
/accent/anti-habits  /:id     анти-привычки (живой таймер дн:ч:мин:сек)
/accent/obstacles             препятствия + контрмеры
/accent/weekly                недельный планер
/accent/lessons               журнал уроков (якорь)
/accent/stats                 очки, уровень, серии, достижения
/accent/supporters            поддержка / круг доверия (MVP — реестр)
/accent/settings              тема, timezone, identity, пауза-режим
```
Каждый экран обрабатывает 4 состояния: **loading** (скелетоны), **empty** (CTA-призыв, не вина), **error** (+ «Повторить»), **loaded**.

## 5. Дашборд (кокпит)
Mobile-first стек, сверху вниз:
1. Хедер: hero-name/уровень + XP-бар.
2. **Чек-ин состояния** (если сегодня не заполнен) — 1 тап по энергии/настроению.
3. «Сегодня»: чеклист задач + % дня + 🔥серия. В recovery — только минимальные версии.
4. **Рекомендации** (`/recommendations`): 1–3 микро-действия под текущее состояние.
5. Неделя: donut % + бары по дням, лучший/худший.
6. Активные цели: 3–5 карточек (progress, прогноз).
7. Просроченные действия / активные Obstacle (препятствия) — с быстрой контрмерой.
8. «Урок-якорь» (если не в ресурсе) — твоя фраза себе.

Desktop ≥1024 — 2 колонки (лево: сегодня+неделя+цели; право: тренды самочувствия (из CheckIn)+серии+достижения+просрочено). Агрегируется одним `GET /accent/dashboard`.

> **Самокомандование (уточнить на 2.10):** «Рекомендации» (п.4, «Сейчас: …») — это директивная команда дня; на финальном дашборде вынести её ближе к ГЕРОЮ экрана (после чек-ина), а не топить в стопке. Дашборд = «делай вот это», не пассивный список.

## 6. Ключевые UX-механики (наши усиления)
- **Минимальная победа на карточке:** вечером, если задача `pending` и серия под угрозой → подсказка «Минимум сегодня: …» (данные уже в задаче, `minVersion`/`ladder.minTarget`).
- **Лесенка вживую:** при `ladder.raised` — мягкий тост «Поднимаем планку: N» (не давление, поздравление). При `lowered` — «Бережём ритм».
- **Анти-привычка:** крупный живой таймер (фронт `setInterval`), circular ring к `targetDays`, кнопка «Сорвался» с подтверждением + поле причины (триггер).
- **Достижение:** модалка `MatDialog` с CSS-конфетти, «Здорово». Звук off по умолчанию.
- **Частичное выполнение:** для quantitative/timed — ввод факта (30 из 50) одним жестом; засчитывается пропорционально, ≥minTarget держит серию.

## 7. Компоненты раздела (свои, SCSS)
ProgressBar (прогресс цели: % + полоса — построено в 2.5), **Chart (`app-chart`) — линейный / area-график на ApexCharts за своим wrapper, тема из CSS-переменных, движок заменяем ([ADR-0055](../../decisions/0055-progress-charts-apexcharts.md))**, XpBar, StreakBadge (🔥N), TaskCard (чекбокс/частичный ввод/swipe-перенос), GoalCard, MicroWinChip, AntiHabitTimer, CheckinSlider (1..10), Heatmap (как GitHub), **AttributeRadar (паучья диаграмма баланса атрибутов — [ADR-0028](../../decisions/0028-accent-timezone-and-domains.md))**, EmptyState (с CTA), Toast. Модалки/диалоги — только через MatDialog. (radar и ProgressDonut для недельного % на дашборде — план поздних волн, ещё не построены; модель под них — сразу).

## 8. Онбординг (волна, но заложить)
Короткий: что важно (1–3 сферы) → первая цель (с подсказкой) → первая привычка (из шаблонов) → достижение «Первый шаг» сразу → мини-тур. Прогрессивное раскрытие сложных фич. Для «Васи» — первая победа за ~5 минут.

## 9. ПДн и приватность в UI
На свободных полях (`whyItMatters`, `note`, identity, причина срыва, цель) — подсказка «не указывай реальные имена/телефоны/адреса» ([ADR-0001](../../decisions/0001-data-minimization-no-pii.md)). Никакой аналитики залипания.
